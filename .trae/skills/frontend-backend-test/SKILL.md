# 前后端测试技能

## 概述

本技能提供了一套完整的前后端测试方案，包括API测试、UI测试、性能测试等，帮助开发者确保前后端系统的稳定性和可靠性。

## 目录结构

```
frontend-backend-test/
├── SKILL.md              # 技能定义文件
├── scripts/              # 测试脚本
├── references/           # 测试文档和参考资料
└── assets/               # 测试资源
```

## 核心功能

### 1. API测试

使用Postman或curl进行API测试，验证后端接口的正确性和稳定性。

### 2. UI测试

使用Playwright进行前端UI测试，验证前端页面的功能和用户体验。

### 3. 性能测试

使用JMeter或Gatling进行性能测试，验证系统在高负载下的表现。

### 4. 集成测试

测试前后端集成的完整性和正确性。

## 使用方法

### API测试

1. 准备测试数据和测试用例
2. 使用Postman或curl发送请求
3. 验证响应结果

### UI测试

1. 编写测试脚本
2. 运行测试
3. 分析测试结果

### 性能测试

1. 配置测试场景
2. 运行性能测试
3. 分析性能报告

## 脚本和工具

### scripts目录

- `api-test.py`: API测试脚本
- `ui-test.py`: UI测试脚本
- `performance-test.py`: 性能测试脚本

### references目录

- `api-endpoints.md`: API接口文档
- `test-cases.md`: 测试用例文档
- `performance-requirements.md`: 性能要求文档

### assets目录

- `test-data/`: 测试数据
- `screenshots/`: 测试截图
- `reports/`: 测试报告

## 最佳实践

1. 编写清晰的测试用例
2. 使用自动化测试工具
3. 定期运行测试
4. 分析测试结果并持续改进

## 常见问题

1. **API测试失败**: 检查API接口是否正常运行，参数是否正确
2. **UI测试失败**: 检查前端页面是否正常加载，元素是否存在
3. **性能测试不达标**: 优化系统性能，检查数据库查询和代码逻辑

## 总结

本技能提供了一套完整的前后端测试方案，帮助开发者确保系统的稳定性和可靠性。通过使用本技能，开发者可以快速发现和解决系统中的问题，提高系统的质量和用户体验。
