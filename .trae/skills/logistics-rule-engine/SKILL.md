# 物流业务规则引擎

## 规则引擎架构

### 1. 规则引擎核心组件
- **规则解析器**：解析规则语法，生成规则对象
- **规则执行器**：执行规则，产生规则执行结果
- **规则存储**：存储规则定义和执行历史
- **规则管理**：管理规则的生命周期

### 2. 规则执行流程
1. 接收规则执行请求
2. 加载相关规则
3. 执行规则引擎
4. 产生执行结果
5. 存储执行历史
6. 返回执行结果

### 3. 规则冲突处理
- **优先级策略**：根据规则优先级处理冲突
- **时间策略**：根据规则创建时间处理冲突
- **条件策略**：根据规则条件的具体程度处理冲突

## 规则定义语法

### 1. 规则结构
```
rule "规则名称"
    when
        条件
    then
        动作
end
```

### 2. 规则条件
- **简单条件**：属性比较
- **复合条件**：AND, OR, NOT
- **嵌套条件**：条件的嵌套组合
- **表达式条件**：使用表达式语言

### 3. 规则动作
- **赋值动作**：设置属性值
- **计算动作**：执行计算操作
- **调用动作**：调用方法或服务
- **通知动作**：发送通知或消息
- **流程动作**：触发业务流程

## 示例

### 1. 订单优先级规则

```
rule "高优先级订单"
    when
        order.priority == "high"
        order.amount > 1000
    then
        order.setProcessingTime("immediate");
        order.setAssignedTo("priority_team");
end
```

### 2. 路由规划规则

```
rule "最优路径选择"
    when
        route.distance < 50
        route.traffic == "light"
    then
        route.setPriority("high");
        route.setEstimatedTime(route.distance / 60);
end
```

### 3. 配送费用规则

```
rule "标准配送费用"
    when
        order.weight <= 5
        order.distance <= 10
    then
        order.setDeliveryFee(10);
end

rule "超重配送费用"
    when
        order.weight > 5
    then
        order.setDeliveryFee(10 + (order.weight - 5) * 2);
end
```
