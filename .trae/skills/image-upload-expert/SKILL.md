# 前后端图片上传专家技能

## 功能概述

本技能提供前后端图片上传的完整测试方案，包括：
1. **本地上传测试**：验证图片是否能成功上传到本地服务器
2. **腾讯云COS上传测试**：验证图片是否能成功上传到腾讯云对象存储
3. **前端上传组件配置**：检查前端上传组件的配置是否正确
4. **后端API实现**：验证后端上传API的实现是否正确
5. **图片显示测试**：验证上传后的图片是否能正确显示

## 测试流程

### 1. 环境准备

1. **后端服务**：确保后端服务已启动，运行在 `http://localhost:8081`
2. **前端服务**：确保前端服务已启动，运行在 `http://localhost:3002`
3. **数据库**：确保数据库已初始化，包含 `order_image` 表
4. **腾讯云COS**：确保腾讯云COS配置正确，包含有效的 `secretId`、`secretKey`、`bucketName` 和 `region`

### 2. 测试步骤

#### 2.1 本地上传测试

1. **打开前端页面**：访问 `http://localhost:3002/`
2. **导航到订单管理**：点击 "订单管理" 菜单
3. **新增订单**：点击 "新增订单" 按钮，填写订单信息并保存
4. **选择图片**：在订单详情页面，点击 "选择文件" 按钮，选择要上传的图片
5. **开始上传**：点击 "开始上传" 按钮
6. **验证上传结果**：
   - 前端应显示 "上传成功" 提示
   - 后端日志应显示上传成功信息
   - 本地上传目录应包含上传的图片文件
   - 已上传图片列表应显示上传的图片

#### 2.2 腾讯云COS上传测试

1. **打开前端页面**：访问 `http://localhost:3002/`
2. **导航到订单管理**：点击 "订单管理" 菜单
3. **新增订单**：点击 "新增订单" 按钮，填写订单信息并保存
4. **选择图片**：在订单详情页面，点击 "选择文件" 按钮，选择要上传的图片
5. **开始上传**：点击 "开始上传" 按钮
6. **验证上传结果**：
   - 前端应显示 "上传成功" 提示
   - 后端日志应显示上传成功信息
   - 腾讯云COS控制台应显示上传的图片
   - 已上传图片列表应显示上传的图片

### 3. 常见问题排查

#### 3.1 前端问题

- **上传按钮禁用**：检查订单是否已保存，确保订单ID和订单编号已生成
- **文件选择失败**：检查文件类型是否为jpg或png，文件大小是否不超过10MB
- **上传失败**：检查前端控制台日志，查看是否有网络错误或其他错误信息

#### 3.2 后端问题

- **API响应失败**：检查后端日志，查看是否有异常信息
- **COS上传失败**：检查COS配置是否正确，包括secretId、secretKey、bucketName和region
- **本地上传失败**：检查上传目录是否存在，是否有写入权限

#### 3.3 数据库问题

- **图片信息未保存**：检查 `order_image` 表是否存在，结构是否正确
- **图片列表为空**：检查数据库中是否有对应订单的图片记录

## 使用方法

1. **运行测试脚本**：
   ```bash
   python scripts/test-upload-local.py
   python scripts/test-upload-cos.py
   python scripts/test-image-display.py
   ```

2. **查看测试结果**：
   - 脚本输出会显示测试是否成功
   - 后端日志会显示详细的上传过程
   - 前端控制台会显示上传状态和错误信息

3. **排查问题**：
   - 根据测试结果和日志信息，定位问题所在
   - 参考常见问题排查指南，解决问题
   - 重新运行测试脚本，验证问题是否已解决

## 注意事项

1. **文件类型**：只支持jpg和png格式的图片
2. **文件大小**：图片大小不能超过10MB
3. **网络连接**：测试腾讯云COS上传需要网络连接
4. **权限**：确保本地上传目录有写入权限
5. **配置**：确保腾讯云COS配置正确，包括secretId、secretKey、bucketName和region
