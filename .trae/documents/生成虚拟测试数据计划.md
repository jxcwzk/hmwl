# 虚拟测试环境数据生成计划

## 目标
创建完整的虚拟测试环境，包含10个客户、3个调度、10个网点、5个司机，以及50个订单用于验证流程。

## 数据结构设计

### 1. 用户表 (user)
| 角色 | 数量 | 用户名格式 | userType |
|------|------|-----------|----------|
| 客户 | 10 | customer1 ~ customer10 | 2 |
| 调度 | 3 | dispatcher1 ~ dispatcher3 | 1 |
| 网点 | 10 | network1 ~ network10 | 4 |
| 司机 | 5 | driver1 ~ driver5 | 3 |

### 2. 客户扩展表 (customer)
- 为每个客户创建10个发货人和10个收货人
- 发货人格式：customer{N}_sender{1-10}
- 收货人格式：customer{N}_receiver{1-10}

### 3. 网点扩展表 (network_point)
- 为每个网点设置地址

### 4. 订单表 (order)
- 50个订单，随机分配给10个客户
- 订单状态分布：
  - status=0, pricingStatus=0: 10个（待付款/待派发）
  - status=1, pricingStatus=1: 10个（待报价）
  - status=2, pricingStatus=2: 15个（已付款/运输中）
  - status=10/11/12: 15个（已完成）

## 实现步骤

### 步骤1：生成用户数据
通过API创建：
- 10个客户账号 (customer1~10)
- 3个调度账号 (dispatcher1~3)
- 10个网点账号 (network1~10)
- 5个司机账号 (driver1~5)

### 步骤2：创建客户扩展数据
通过API为customer1创建20个联系人（10发货+10收货），重复10次

### 步骤3：创建网点扩展数据
通过API为10个网点创建扩展信息

### 步骤4：创建50个订单
通过API创建50个订单，随机分配：
- 客户ID: 1~10
- 发货人/收货人: 从该客户的联系人中随机选择
- businessUserId: 1~3
- networkPointId: 1~10（部分订单）

### 步骤5：更新订单状态
通过API更新订单状态，确保覆盖所有流程状态

### 步骤6：验证数据
- 查询各角色订单列表
- 确认数据串联正确

## 执行命令脚本

```bash
# 客户1-10
curl -X POST http://localhost:8081/api/user/register ...

# 调度1-3
curl -X POST http://localhost:8081/api/user/register ...

# 网点1-10
curl -X POST http://localhost:8081/api/user/register ...

# 司机1-5
curl -X POST http://localhost:8081/api/user/register ...
```

### 创建订单
通过循环调用订单创建API，生成50个订单

- 3个调度账号可登录
- 10个网点账号可登录
