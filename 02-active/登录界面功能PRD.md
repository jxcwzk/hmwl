## 1. Overview

### 需求描述
在Vue管理后台前端和微信小程序端分别添加用户登录功能，实现统一的身份认证系统。

### 目标用户
- 系统管理员
- 业务客户
- 司机
- 网点工作人员

### 解决的问题
- 现有系统无登录功能，存在安全隐患
- 无法控制用户访问权限
- 用户角色混乱，无法区分不同业务权限

## 3. Functional Requirements

### 3.1 Vue管理后台登录

#### FR-01: 登录页面
- 登录页面采用iOS风格设计，与现有系统保持一致
- 页面包含：Logo、标题、用户名输入框、密码输入框、登录按钮
- 记住密码复选框（可选功能）
- 错误提示信息显示区域

#### FR-02: 登录验证
- 用户名密码验证
- 输入格式校验（不能为空）
- 登录失败显示错误信息
- 登录成功后保存Token到本地存储

#### FR-03: 路由守卫
- 未登录用户访问需要认证的页面时，自动跳转到登录页
- 登录页面不需要认证即可访问
- 登录成功后自动跳转到首页或之前访问的页面

#### FR-04: 角色权限控制
- 管理员登录后看到完整菜单（用户、司机、网点、车辆、订单、财务、统计、路线、发票、业务客户）
- 客户登录后仅看到：我的订单、网点查询
- 司机登录后仅看到：我的任务、网点查询
- 网点工作人员登录后看到：订单管理、网点查询

### 3.2 微信小程序登录

#### FR-05: 微信授权登录
- 显示微信授权登录按钮
- 用户授权后获取微信用户信息
- 调用后端接口换取系统Token

#### FR-06: Token管理
- Token存储到本地存储
- 每次请求自动携带Token
- Token过期自动跳转登录页

#### FR-07: 退出登录
- 清除本地存储的Token和用户信息
- 跳转到登录页

### 3.3 后端登录API

#### FR-08: 用户名密码登录接口
- 请求路径：POST /api/user/login
- 请求参数：username, password
- 返回：token, userInfo, roles

#### FR-09: 微信登录接口
- 请求路径：POST /api/user/wxLogin
- 请求参数：code（微信授权码）
- 返回：token, userInfo, roles

#### FR-10: Token验证接口
- 请求路径：GET /api/user/info
- 请求头：Authorization: Bearer {token}
- 返回：用户信息、权限列表

#### FR-11: 退出登录接口
- 请求路径：POST /api/user/logout
- 处理：清除Token会话

## 4. Acceptance Criteria

### 4.1 Vue管理后台登录

| 编号 | 验收标准 |
|------|----------|
| AC-01 | 登录页面在未登录状态下可以正常访问 |
| AC-02 | 输入正确的用户名密码后成功登录并跳转到首页 |
| AC-03 | 输入错误的用户名密码后显示错误提示"用户名或密码错误" |
| AC-04 | 用户名为空时显示"请输入用户名" |
| AC-05 | 密码为空时显示"请输入密码" |
| AC-06 | 登录成功后Token保存到localStorage |
| AC-07 | 刷新页面后保持登录状态 |
| AC-08 | 不同角色用户看到不同的菜单 |
| AC-09 | 点击退出登录后清除登录状态并跳转到登录页 |
| AC-10 | 未登录状态下访问其他页面自动跳转到登录页 |

### 4.2 微信小程序登录

| 编号 | 验收标准 |
|------|----------|
| AC-11 | 未登录状态下打开小程序弹出登录授权页面 |
| AC-12 | 点击微信授权按钮成功登录 |
| AC-13 | 登录成功后Token保存到本地 |
| AC-14 | 再次打开小程序自动保持登录状态 |
| AC-15 | 点击退出登录清除登录状态 |

### 4.3 后端API

| 编号 | 验收标准 |
|------|----------|
| AC-16 | 用户名密码登录返回有效Token |
| AC-17 | 微信登录返回有效Token |
| AC-18 | 携带有效Token请求用户信息成功 |
| AC-19 | 无Token或Token过期返回401 |
| AC-20 | 退出登录后Token失效 |

## 5. Edge Cases

| 编号 | 边界情况 | 处理方案 |
|------|----------|----------|
| EC-01 | 网络请求失败 | 显示"网络错误，请稍后重试" |
| EC-02 | Token过期 | 自动跳转登录页，提示"登录已过期" |
| EC-03 | 微信授权被拒绝 | 显示"需要授权才能使用"提示 |
| EC-04 | 用户被禁用 | 显示"账号已被禁用，请联系管理员" |
| EC-05 | 密码连续错误5次 | 锁定账号15分钟 |
| EC-06 | 微信code已使用 | 提示"操作过快，请重试" |

## 9. 技术方案

### 9.1 前端技术方案
- Vue 3 + Vue Router 4
- Token存储：localStorage
- 请求拦截器：axios添加Token
- 路由守卫：router.beforeEach

### 9.2 后端技术方案
- JWT Token（可配置过期时间，默认24小时）
- 密码加密：BCrypt
- Token刷新：Refresh Token（可选）

### 9.3 数据库设计
- 用户表已有role字段，扩展roles数组
- 可复用现有user表
