# 订单页面货物照片卡片

## 1. Overview
- **需求描述**：在订单创建/编辑对话框中添加"货物照片"卡片，与现有的发货单照片、回单照片保持一致的布局和交互形式
- **目标用户**：物流订单操作人员
- **解决的问题**：完善订单详情展示，支持货物照片上传和管理

## 2. User Stories
- 作为订单操作人员，我希望在创建/编辑订单时能上传货物照片，以便记录货物实际状态
- 作为订单操作人员，我希望货物照片能持久化保存，下次打开订单还能看到
- 作为订单操作人员，我希望货物照片可以删除和预览

## 3. Functional Requirements
- **FR1**: 在订单对话框的图片区域添加"货物照片"卡片
- **FR2**: 货物照片卡片布局与现有发货单照片、回单照片保持一致
- **FR3**: 卡片从左到右依次为：货物照片、发货单照片、回单照片
- **FR4**: 支持上传新图片（拖拽或选择文件）
- **FR5**: 支持预览已上传的图片（点击放大）
- **FR6**: 支持删除已上传的图片
- **FR7**: 图片存储到腾讯云COS，URL持久化到数据库
- **FR8**: 重新打开订单时能加载并显示之前上传的货物照片

## 4. Acceptance Criteria
- **AC1**: 订单对话框中图片区域显示三个卡片，从左到右依次为：货物照片、发货单照片、回单照片
- **AC2**: 每个卡片都显示已上传图片数量Badge
- **AC3**: 可以上传JPG/PNG格式的图片，单个文件不超过10MB
- **AC4**: 上传后图片立即显示在卡片中
- **AC5**: 点击图片可以预览（放大）
- **AC6**: 可以删除已上传的图片
- **AC7**: 关闭对话框后重新打开，货物照片仍然显示
- **AC8**: 图片上传到腾讯云COS，数据库存储图片URL

## 5. Edge Cases
- **EC1**: 上传非图片格式文件时提示错误
- **EC2**: 上传超过10MB的文件时提示错误
- **EC3**: 未保存订单时提示先保存订单再上传图片
