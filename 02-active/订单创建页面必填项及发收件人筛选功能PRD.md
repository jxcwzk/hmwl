# 订单创建页面必填项及发收件人筛选功能 PRD

> **状态：已完成** ✅

## 1. Overview

### 需求描述
前端订单创建页面需要设置必填项校验，防止重要物流信息遗漏。同时需要确保发件人和收件人信息基于所选业务用户进行筛选关联，确保不同业务用户对应不同的发件人和收件人资源。

### 目标用户
- 物流订单操作人员
- 网点工作人员

### 解决的问题
1. 防止订单创建时重要信息遗漏导致后续物流问题
2. 确保发件人/收件人只能选择当前业务用户关联的资源，避免数据混乱
3. 提升订单数据的完整性和准确性

---

## 2. User Stories

| 序号 | User Story |
|------|------------|
| US1 | 作为订单操作人员，我希望在创建订单时系统强制要求填写关键信息（业务用户、发件人、收件人），以防止遗漏重要物流信息 |
| US2 | 作为订单操作人员，我希望选择业务用户后，发件人和收件人下拉列表只显示该业务用户关联的人员，以便快速准确选择正确的发件人/收件人 |
| US3 | 作为订单操作人员，我希望在未填写必填项时无法提交订单，系统给出清晰的错误提示 |

---

## 3. Functional Requirements

### 3.1 必填项设置

| 字段名称 | 字段标识 | 必填 | 说明 |
|----------|----------|------|------|
| 业务用户 | businessUserId | 是 | 订单来源，最重要的关联 |
| 发件人 | senderId | 是 | 发件人信息 |
| 发件人姓名 | senderName | 是 | 发件人姓名 |
| 发件人电话 | senderPhone | 是 | 发件人联系方式 |
| 发件人地址 | senderAddress | 是 | 发件人地址 |
| 收件人 | recipientId | 是 | 收件人信息 |
| 收件人姓名 | receiverName | 是 | 收件人姓名 |
| 收件人电话 | receiverPhone | 是 | 收件人联系方式 |
| 收件人地址 | receiverAddress | 是 | 收件人地址 |
| 货物名称 | goodsName | 是 | 货物描述 |
| 重量 | weight | 是 | 货物重量(kg) |
| 起始网点 | startNetworkId | 是 | 订单起始网点 |

### 3.2 发件人/收件人筛选功能

| 功能描述 | 详细说明 |
|----------|----------|
| 业务用户切换联动 | 选择业务用户后，自动清空已选的发件人和收件人，重新加载该业务用户关联的发件人/收件人列表 |
| 发件人列表加载 | 根据选定的 businessUserId，调用后端 API 获取 type=0（发件人）的 BusinessCustomer 列表 |
| 收件人列表加载 | 根据选定的 businessUserId，调用后端 API 获取 type=1（收件人）的 BusinessCustomer 列表 |
| 切换业务用户时清空选择 | 当业务用户变更时，需要清空：senderId、senderName、senderPhone、senderAddress、recipientId、receiverName、receiverPhone、receiverAddress |

### 3.3 表单验证逻辑

| 验证场景 | 验证规则 |
|----------|----------|
| 必填项校验 | 提交订单前检查所有必填字段，若为空则阻止提交并高亮对应字段 |
| 手机号格式校验 | 发件人/收件人电话需符合11位手机号格式 |
| 业务用户必选 | 必须先选择业务用户，才能选择发件人和收件人 |

## 4. Acceptance Criteria

### AC1: 必填项校验
- [ ] 订单创建页面打开时，所有必填字段旁边显示红色星号标识
- [ ] 点击提交按钮时，若存在未填写的必填项，页面滚动到第一个错误位置并显示错误提示
- [ ] 错误提示文案清晰，如"请选择业务用户"、"请填写发件人电话"等

### AC2: 发件人/收件人筛选
- [ ] 选择业务用户后，发件人下拉列表显示该业务用户关联的所有发件人（type=0）
- [ ] 选择业务用户后，收件人下拉列表显示该业务用户关联的所有收件人（type=1）
- [ ] 切换业务用户时，发件人和收件人下拉列表自动更新为新业务用户关联的列表
- [ ] 切换业务用户时，之前选中的发件人和收件人信息自动清空

### AC3: 数据填充
- [ ] 选择发件人后，自动填充：senderName、senderPhone、senderAddress
- [ ] 选择收件人后，自动填充：receiverName、receiverPhone、receiverAddress

### AC4: 用户体验
- [ ] 必填项校验在客户端完成，无需等待服务器响应
- [ ] 错误提示在对应字段下方显示，不遮挡其他内容

## 5. Edge Cases

| 场景 | 处理方式 |
|------|----------|
| 业务用户下无发件人 | 显示空列表，提示"该业务用户暂无发件人信息，请先添加" |
| 业务用户下无收件人 | 显示空列表，提示"该业务用户暂无收件人信息，请先添加" |
| 切换业务用户时发件人已填写 | 自动清空发件人相关字段 |
| 网络请求失败 | 显示错误提示，保留已填写的信息 |
| 页面刷新 | 重置表单，所有字段恢复为空 |

## 9. 技术实现提示

### 后端 API（已存在）
```
GET /business-customer/listByBusinessUserIdAndType/{businessUserId}/{type}
```
- businessUserId: 业务用户ID
- type: 0-发件人, 1-收件人

### 前端组件
- 文件：[Order.vue](file:///Users/jiangxiaochun/Desktop/hmwl/hmwl/frontend/src/views/Order.vue)
- 需要修改的函数：

  - `handleSubmit`: 添加必填项校验
  - 添加校验函数：`validateForm`

### 数据关系
```
BusinessUser (业务用户)
    │
    ├── 1:N ──► BusinessCustomer (type=0, 发件人)
    │
    └── 1:N ──► BusinessCustomer (type=1, 收件人)
```
