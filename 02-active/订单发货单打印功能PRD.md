# 订单发货单打印功能 PRD

## Status
- [x] 已完成（2026-03-12）

## 1. Overview
- **需求描述**: 为每条订单提供打印发货单功能，输出 PDF 格式，用于回单凭证拍照上传
- **目标用户**: 物流公司工作人员
- **解决的问题**: 传统手写发货单效率低、易出错，需要快速生成标准化电子发货单

## 2. User Stories
- **Story 1**: 作为物流工作人员，我希望在订单详情页面点击"打印发货单"按钮，生成并下载 PDF 文件，以便快速获取标准发货单
- **Story 2**: 作为物流工作人员，我希望发货单包含二维码、订单编号、收发货人信息、货物信息等关键数据，以便拍照上传作为回单凭证
- **Story 3**: 作为物流工作人员，我希望发货单包含固定的公司 Logo 和发件人信息，以提升公司品牌形象

## 3. Functional Requirements

### 3.1 发货单展示内容

#### 发件人信息（固定）
| 字段 | 值 |
|------|-----|
| 发件人 | 郭红美 |
| 联系电话 | 15315162177 |
| 联系地址 | 靖江红美物流 |

#### 收件人信息（从订单获取）
| 字段 | 说明 |
|------|------|
| 收件人姓名 | order.receiverName / order.recipientName |
| 收件人电话 | order.receiverPhone |
| 收件人地址 | order.receiverAddress |

#### 订单信息
| 字段 | 说明 |
|------|------|
| 二维码 | 包含订单编号的二维码（复用现有 QRCode 实现） |
| 订单编号 | order.orderNo |
| 订单类型 | order.orderType（0=零担，1=整车） |

#### 货物信息
| 字段 | 说明 |
|------|------|
| 货物名称 | order.goodsName |
| 数量 | order.quantity |
| 重量 | order.weight（单位：kg） |
| 体积 | order.volume（单位：m³） |
| 备注 | order.remark |

#### 品牌元素
- 红美物流 Logo

### 3.2 交互流程
1. 用户在订单列表点击"编辑"打开订单详情对话框
2. 在订单详情对话框底部添加"打印发货单"按钮
3. 点击按钮后，前端生成 PDF 文件并自动下载
4. 文件命名格式：`发货单_{订单编号}.pdf`

### 3.3 PDF 布局要求
- A4 尺寸（210mm × 297mm）
- 竖向布局
- 简洁清晰，便于拍照上传
- 包含必要的分隔线和区域划分

## 4. Acceptance Criteria

### 4.1 功能验收
- [ ] 订单详情对话框显示"打印发货单"按钮
- [ ] 点击按钮生成并下载 PDF 文件
- [ ] PDF 包含所有要求的字段信息
- [ ] 发件人信息为固定值（郭红美）
- [ ] 收件人信息正确从订单数据中获取
- [ ] 二维码正确生成（包含订单编号）
- [ ] 红美物流 Logo 正确显示

### 4.2 视觉验收
- [ ] PDF 布局整洁，文字清晰
- [ ] 各项信息分区明确
- [ ] Logo 正确显示，无变形
- [ ] 二维码清晰可扫描

### 4.3 数据验收
- [ ] 无报价/费用信息（总价、运费等不显示）
- [ ] 订单编号正确显示
- [ ] 订单类型正确转换（0=零担，1=整车）
- [ ] 货物信息完整（名称、数量、重量、体积）
- [ ] 备注信息正确显示

## 5. Edge Cases
- **订单无收件人信息**: 显示"暂无"或空值
- **订单无货物信息**: 显示"暂无"或空值
- **订单编号为空**: 提示"订单未保存，无法生成发货单"
- **Logo 图片加载失败**: 显示文字"红美物流"作为备选

## 9. Technical Notes

### 推荐技术方案
- **PDF 生成库**: html2pdf.js（将 HTML 转换为 PDF）
- **二维码库**: 复用现有 qrcode（已在项目中使用）
- **Logo 资源**: 需要提供红美物流 Logo 图片文件

### 实现思路
1. 创建一个隐藏的发货单 HTML 模板
2. 使用 Vue 的响应式数据填充模板
3. 使用 html2pdf.js 将 HTML 渲染为 PDF
4. 触发浏览器下载
