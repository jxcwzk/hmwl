# 红美物流业务运行逻辑规范

## Why

为了规范红美物流系统的业务流程，明确五个核心角色（客户、调度、网点、司机、财务）的职责分工、业务流程、数据流转和状态机定义，确保系统业务逻辑清晰、可执行、可追踪。

## What Changes

- 定义红美物流五个核心角色的完整业务运行逻辑
- 新增财务角色，处理订单结算、开票等财务相关业务
- 规范订单从创建到完成的完整生命周期流程
- 明确角色间的交互关系和数据流转
- 完善状态机定义，支持业务追踪

## Impact

- Affected specs: 客户下单流程、调度派单流程、网点操作流程、司机配送流程、财务结算流程
- Affected code: 订单状态管理、角色权限控制、业务流程API

### Requirement: 订单生命周期状态机

#### 主状态 (status) 流转

| status | 状态名 | 触发角色 | 说明 |
|--------|--------|---------|------|
| 0 | 订单创建 | 客户 | 客户提交新订单 |
| 1 | 已派发比价 | 调度 | 调度派发给网点询价 |
| 4 | 报价已推送 | 调度 | 报价已推送给客户 |
| 5 | 价格已确认 | 客户 | 客户确认运输价格 |
| 9 | 网点已收货 | 网点 | 网点确认收货入库 |
| 10 | 网点已发货 | 网点 | 网点拼车后发出 |
| 11 | 配送中 | 配送司机 | 配送司机运输中 |
| 12 | 已到达 | 配送司机 | 货物已到达目的地 |
| 13 | 已签收 | 客户 | 客户确认签收 |
| 14 | 已完成 | 财务 | 订单结算完成 |

### Requirement: 财务角色业务逻辑

#### 财务职责

财务角色负责订单完成后的结算和开票工作：

| 业务 | 操作 | 触发时机 | 说明 |
|------|------|---------|------|
| 查看待结算订单 | GET /finance/pending | 客户签收后 | 查看已签收待结算的订单 |
| 创建结算记录 | POST /finance/settle | 财务手动 | 生成结算单，计算费用 |
| 确认结算 | POST /finance/confirm | 财务手动 | 确认结算信息 |
| 开票 | POST /finance/invoice | 结算确认后 | 开具发票 |
| 查看结算历史 | GET /finance/history | 任意 | 查看历史结算记录 |

#### 结算流程

```
客户签收 ──► 财务查看待结算 ──► 创建结算单 ──► 确认结算 ──► 开票 ──► 订单完成
   │              │               │            │           │           │
   ▼              ▼               ▼            ▼           ▼           ▼
status=13    结算待处理      生成结算单    结算确认     开具发票    status=14
              订单列表        计算费用      财务审核     发票记录    已完成
```

#### 结算费用计算

| 费用项 | 计算方式 | 说明 |
|--------|---------|------|
| 基础运费 | 定价报价 | 网点报价的运输费用 |
| 上门取货费 | 固定/按重量 | 提货司机上门取货费用 |
| 配送费 | 固定/按距离 | 配送司机送货上门费用 |
| 保价费 | 货值×费率 | 可选的货物保险费用 |
| 其他费用 | 实报实销 | 装卸费、停车费等 |
| **合计** | 累加 | 最终结算金额 |

### Requirement: 司机角色区分

系统中有两种不同的司机角色，分别由不同的角色管理：

#### 提货司机 (Pickup Driver) - driver1

| 属性 | 说明 |
|------|------|
| 归属 | 系统/调度管理 |
| 分配者 | 调度（admin） |
| 职责 | 从客户处提货 → 运送到网点 |
| 任务结束 | 货物送达网点，网点确认收货后 |
| 后续操作 | 不参与，由网点配送司机接手 |

#### 配送司机 (Delivery Driver) - driver2/driver3等

| 属性 | 说明 |
|------|------|
| 归属 | 网点管理（network维护） |
| 分配者 | 网点（network） |
| 职责 | 从网点提货 → 配送到客户 |
| 任务结束 | 货物送达客户，客户签收后 |
| 后续操作 | 任务完成，由财务进行结算 |
