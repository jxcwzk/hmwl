## 1. Overview

### 需求描述
将微信小程序从微信云开发迁移到调用后端Java API，实现小程序、VUE前端、后端云服务三者数据互通，同时保留云开发部分功能（图片上传）。优先对接订单模块，并实现微信自动登录功能。

### 目标用户
- 小程序终端用户（客户、司机）
- 红美物流后台管理员

### 解决的问题
- 小程序数据与Vue前端不一致的问题
- 手动管理用户身份的繁琐
- 三端数据统一，方便后续维护

## 3. Functional Requirements

### 3.1 登录与身份管理

#### FR1: 微信自动登录
- 小程序调用wx.login获取code
- 将code发送到后端Java API换取openid
- 后端根据openid查找或创建用户记录
- 返回用户信息（包含身份状态）

#### FR2: 身份状态查询
- 用户登录后，前端查询用户身份状态
- 状态包括：未分配、待审核、已分配（客户/司机/管理员）
- 未分配身份时，提示用户联系管理员

#### FR3: 后台用户管理（VUE端）
- 管理员查看所有小程序注册用户
- 管理员为用户指派身份角色
- 支持角色：客户、司机、网点管理员、系统管理员

#### FR4: 小程序端身份展示
- 用户登录后显示个人中心
- 展示当前身份状态
- 未分配身份时，显示提示信息

### 3.2 订单模块对接

#### FR5: 订单列表API对接
- 小程序调用后端 `/api/order/list` 获取订单列表
- 实现分页查询
- 对接现有订单筛选功能

#### FR6: 订单详情API对接
- 小程序调用后端 `/api/order/{id}` 获取详情
- 展示订单完整信息

#### FR7: 创建订单API对接
- 小程序调用后端 `/api/order/add` 创建订单
- 支持订单相关图片上传

### 3.3 其他模块对接（可选扩展）

#### FR8: 司机列表对接
- 小程序调用后端 `/api/driver/list` 获取司机列表

#### FR9: 网点列表对接
- 小程序调用后端 `/api/network/list` 获取网点列表

#### FR10: 车辆列表对接
- 小程序调用后端 `/api/vehicle/list` 获取车辆列表

### 3.4 基础设施

#### FR11: 环境配置
- 开发环境：baseURL = `http://localhost:8081`
- 生产环境：baseURL = 云端服务器地址
- 通过配置文件切换

#### FR12: 云函数保留
- 保留图片上传相关云函数
- 图片存储继续使用腾讯云COS

## 5. Edge Cases

| Case | 处理方式 |
|------|----------|
| 用户频繁点击登录 | 防止重复提交，显示loading状态 |
| 网络请求失败 | 显示错误提示，提供重试按钮 |
| Token过期 | 引导用户重新登录 |
| 管理员未指派身份 | 提示"请等待管理员分配身份" |
| 后端服务未启动 | 提示"服务不可用，请稍后重试" |

## 9. 技术方案补充

### 9.1 API BaseURL
```
开发环境: http://localhost:8081
生产环境: 配置在 application-prod.yml
```

### 9.2 登录流程
```
小程序 wx.login() → 后端 /api/user/wxLogin → 返回用户信息+Token
```

### 9.3 需新增的API
| API | 方法 | 说明 |
|-----|------|------|
| /api/user/wxLogin | POST | 微信登录 |
| /api/user/info | GET | 获取用户信息 |
| /api/user/list | GET | 用户列表（后台） |
| /api/user/updateRole | POST | 更新用户角色（后台） |

### 9.4 需保留的云函数
- 图片上传云函数（wx.cloud.uploadFile）
