## 1. Overview

### 需求描述
优化线路管理和网点管理页面，实现以下功能：
1. 线路与网点是多对多关系（一条线路经过多个网点）
2. 客户下单后，根据起止城市自动匹配线路
3. 基于匹配到的线路，展示沿线网点供调度选择
4. 调度可优先选择线路网点，也可选择其他网点

### 目标用户
- 调度管理员

### 解决的问题
当前线路和网点缺乏关联，调度派发时无法基于线路智能推荐网点。

## 2. User Stories

### Story 1: 线路管理
**作为** 调度管理员
**我希望** 管理线路及其经过的网点
**以便** 维护物流线路的网点覆盖

### Story 2: 智能匹配
**作为** 调度管理员
**我希望** 输入起止城市后自动匹配线路
**以便** 快速找到合适的物流线路

### Story 3: 网点推荐
**作为** 调度管理员
**我希望** 查看沿线网点列表并优先推荐
**以便** 高效选择合适的网点派发报价

## 3. Functional Requirements

### 3.1 数据结构

#### 新建关联表 `route_network_point`
| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键 |
| route_id | Long | 线路ID |
| network_point_id | Long | 网点ID |
| sequence | Integer | 在线路上的顺序（可选） |

#### 修改 Route.java
- 保留字段：id, startCity, destinationCity, basePrice, pricePerKg, transitDays, createTime, updateTime
- **移除**：networkPointId（一对多改为多对多）
- **新增**：与 NetworkPoint 的多对多关系

### 3.2 后端 API

#### 线路管理
| API | 方法 | 说明 |
|-----|------|------|
| `/api/route/list` | GET | 获取所有线路 |
| `/api/route/{id}` | GET | 获取线路详情（含沿线网点） |
| `/api/route/{id}/networks` | GET | 获取线路关联的网点列表 |
| `/api/route/{id}/networks` | POST | 添加网点到线路 |
| `/api/route/{id}/networks/{networkId}` | DELETE | 从线路移除网点 |
| `/api/route/match` | GET | 根据起止城市匹配线路 |

#### 请求参数
```
GET /api/route/match?startCity=上海&destinationCity=北京
```

#### 响应
```json
{
  "id": 1,
  "startCity": "上海",
  "destinationCity": "北京",
  "basePrice": 100,
  "pricePerKg": 0.5,
  "transitDays": 3,
  "networks": [
    {"id": 1, "name": "上海网点", "city": "上海"},
    {"id": 2, "name": "南京网点", "city": "南京"},
    {"id": 3, "name": "北京网点", "city": "北京"}
  ]
}
```

### 3.3 前端页面

#### 3.3.1 线路管理页面 (Route.vue)

**功能列表**：
1. 线路列表展示（起点城市、终点城市、基准价）
2. 线路详情抽屉
   - 显示基本信息
   - 显示关联网点列表
   - 添加/移除网点功能
3. 按起止城市搜索
4. 新建/编辑/删除线路

**网点选择**：
- 弹出网点选择对话框
- 显示所有可用网点
- 支持多选
- 已关联网点显示选中状态

#### 3.3.2 网点管理页面 (NetworkPoint.vue)

**功能列表**：
1. 网点列表展示
2. 显示每个网点所在的线路列表
3. 新建/编辑/删除网点

#### 3.3.3 调度管理页面 (Dispatch.vue)

**修改"选择网点派发"功能**：
1. 显示订单的起止城市信息
2. 调用 `/api/route/match` 自动匹配线路
3. 优先显示沿线网点（打标签"沿线推荐"）
4. 也可展开显示全部网点供选择
5. 选择网点后派发报价

**UI 设计**：
```
┌─────────────────────────────────────────┐
│ 订单信息                                │
│ 发货地：上海  →  收货地：北京            │
├─────────────────────────────────────────┤
│ 匹配线路：沪宁线 [沿线网点 3 个]         │
│ ┌─────────────────────────────────────┐ │
│ │ ● 上海网点 [沿线推荐] ✓已选          │ │
│ │ ○ 南京网点 [沿线推荐]                │ │
│ │ ○ 武汉网点                          │ │
│ │ ○ 北京网点 [沿线推荐]                │ │
│ └─────────────────────────────────────┘ │
│                                         │
│ [取消]                    [确认派发]     │
└─────────────────────────────────────────┘
```

## 4. Acceptance Criteria

### 4.1 线路-网点关联
- [ ] 可以创建线路
- [ ] 可以为线路添加多个网点
- [ ] 可以从线路移除网点
- [ ] 线路详情显示所有关联网点

### 4.2 城市匹配
- [ ] 根据起止城市返回匹配线路
- [ ] 返回结果包含沿线网点列表
- [ ] 无匹配线路时返回空列表

### 4.3 调度派发
- [ ] 派发时显示起止城市
- [ ] 自动匹配并显示沿线网点
- [ ] 沿线网点有"推荐"标识
- [ ] 可选择其他非沿线网点

## 5. Edge Cases

| 场景 | 处理方式 |
|------|----------|
| 起止城市无匹配线路 | 提示"无可用线路"，允许手动选择 |
| 线路无关联网点 | 提示"该线路暂无网点"，允许跳过 |
| 多个线路匹配 | 返回所有匹配线路，用户选择 |
| 城市名称不一致 | 使用模糊匹配（如"上海市"≈"上海"） |
