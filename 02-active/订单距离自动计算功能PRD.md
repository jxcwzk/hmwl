# 订单距离自动计算功能 PRD

> **状态：已完成** ✅

## 1. Overview

- **需求名称**: 订单距离自动计算功能
- **需求描述**: 在创建/编辑订单时，根据发货人地址和收货人地址自动计算两点之间的距离，自动填充到距离字段，无需手动填写，方便后续价格计算
- **目标用户**: 物流系统操作人员、客服人员
- **解决的问题**: 手动填写距离容易出错，自动化计算提高准确性

## 2. User Stories

- **Story 1**: 作为客服人员，我希望在创建订单时自动计算发货人和收货人地址之间的距离，这样我不需要手动查询和填写距离
- **Story 2**: 作为财务人员，我希望订单的距离数据准确无误，这样运费计算才能正确
- **Story 3**: 作为系统管理员，我希望距离计算支持多种模式（地图API/直线距离），以适应不同网络环境

## 3. Functional Requirements

### 3.1 距离计算触发
- 当订单的发货地址或收货地址发生变化时，自动触发距离计算
- 支持手动点击"计算距离"按钮主动触发
- 计算完成后自动填充距离字段

### 3.2 地址获取
- **仅从订单的直接地址字段获取**：
  - 发货人地址：`sender_address`
  - 收货人地址：`receiver_address`
- 不再从关联网点获取地址

### 3.3 距离计算模式
- **模式一**: 地图API计算（推荐）- 调用腾讯地图/高德地图API计算实际路线距离
- **模式二**: 直线距离计算 - 根据经纬度计算球面直线距离（备选方案）

### 3.4 距离数据存储
- 计算结果存储到订单的 `distance` 字段

### 3.5 错误处理
- 当地址信息不完整时，提示用户补充地址信息
- 当地图API调用失败时，自动切换到直线距离计算
- 当所有计算都失败时，允许手动填写距离

## 4. Acceptance Criteria

- [x] 在订单创建页面，填写发货人和收货人地址后，距离自动填充
- [x] 修改发货或收货地址后，距离自动重新计算
- [x] 距离计算失败时，系统给出明确提示
- [x] 手动修改距离后，不会被自动覆盖（除非地址再次变化）
- [x] 直线距离计算作为兜底方案始终可用

## 5. Edge Cases

- **Case 1**: 发货地址或收货地址为空 → 提示用户补充地址，不计算距离
- **Case 2**: 地址信息不完整 → 尝试计算或提示补充
- **Case 3**: 地图API服务不可用 → 自动切换到直线距离计算
- **Case 4**: 两地址相同 → 距离设为0
- **Case 5**: 订单已保存，手动修改距离 → 保留手动值，下次地址变更时重新计算

## 6. Priority

**高优先级**

理由：
- 是价格计算的前置功能
- 自动化数据录入，减少人工错误
- 影响核心业务流程

## 9. 技术方案建议

### 9.1 直线距离计算（兜底方案）
使用 Haversine 公式根据两点经纬度计算球面距离

### 9.2 地址格式
- 支持两种格式：
  1. 经纬度格式：`39.9042,116.4074`（纬度,经度）
  2. 文字地址：`北京市朝阳区xxx`（需要地图API支持）

### 9.3 配置项
```
distance:
  mode: straight  # auto: 自动选择, map: 强制地图API, straight: 强制直线距离
  map:
    provider: tencent  # tencent: 腾讯地图, amap: 高德地图
    api-key: ${MAP_API_KEY}
```
