# 图片上传大小限制与压缩功能

## 1. Overview

- **需求描述**：在图片上传到腾讯云COS前进行大小判断，大于2MB的图片自动压缩后再上传，控制CDN流量费用
- **目标用户**：运营人员、财务人员
- **解决的问题**：解决腾讯云COS图片预览产生的CDN流量费用过高问题

## 2. User Stories

- **Story 1**: 作为运营人员，我希望上传小于2MB的图片时直接上传，这样加载速度快
- **Story 2**: 作为运营人员，我希望上传大于等于2MB的图片时自动压缩，这样不影响CDN费用
- **Story 3**: 作为财务人员，我希望上传的发货单/回单图片体积适中，这样查看方便且节省成本

## 3. Functional Requirements

- **FR-001**: 图片大小判断 - 上传图片前判断文件大小
- **FR-002**: 小图直传 - 小于2MB的图片直接上传到COS
- **FR-003**: 大图压缩 - 大于等于2MB的图片自动压缩至小于2MB后再上传
- **FR-004**: 质量控制 - 压缩质量保持70%
- **FR-005**: 格式支持 - 支持 jpg、png、gif、bmp、webp 格式
- **FR-006**: 服务覆盖 - 发货单图片和回单图片上传均应用此规则

## 4. Acceptance Criteria

- **AC-001**: 上传1.5MB的图片，文件直接上传，不进行压缩处理
- **AC-002**: 上传5MB的jpg图片，压缩后文件小于2MB，上传成功
- **AC-003**: 上传3MB的png图片，压缩后文件小于2MB，上传成功
- **AC-004**: 上传10MB的图片，经过多次压缩后文件小于2MB，上传成功
- **AC-005**: 压缩后的图片仍然清晰可读，70%质量肉眼观察无明显失真

## 5. Edge Cases

- **EC-001**: 图片已经是极小尺寸（如100KB），直接上传
- **EC-002**: 压缩后仍大于2MB（极少见），继续降低质量直到可接受或达到最低质量阈值
- **EC-003**: 不支持的图片格式，应返回错误提示

## 6. Priority

- **High** - 直接影响CDN流量费用控制
