## 1. Overview

**需求描述**：财务结算界面与订单模块联动，支持按业务用户和时间段筛选，提供一键开票功能

**目标用户**：财务人员、公司管理层

**解决的问题**：

- 订单完成后自动生成结算记录
- 便于财务人员按条件查询结算记录
- 简化开票流程，提高工作效率

## 2. User Stories

| 角色   | 场景      | 期望                     |
| ---- | ------- | ---------------------- |
| 财务人员 | 回单拍照上传后 | 财务结算列表自动新增一条结算记录       |
| 财务人员 | 查看结算记录时 | 可以按业务用户筛选，查看该客户的所有结算记录 |
| 财务人员 | 查看结算记录时 | 可以按时间段筛选，查看特定期间的结算记录   |
| 财务人员 | 录入结算金额时 | 系统给出基于30%利润率的推荐客户报价    |
| 财务人员 | 录入结算金额时 | 可以手动调整最终结算金额           |
| 财务人员 | 开票时     | 可以一键生成发票               |

## 3. Functional Requirements

### 3.1 订单联动（结算记录自动生成）

**触发条件：订单完成（回单拍照上传）**

- 监听订单状态变化
- 当 order\_status = completed（回单拍照上传）时，自动创建结算记录
- 记录订单编号、业务用户、订单金额（网点成本）
- 不在订单支付完成时生成结算记录

**结算记录内容**：

| 字段     | 说明                         |
| ------ | -------------------------- |
| 结算ID   | 自动生成，唯一标识                  |
| 订单编号   | 关联的订单编号                    |
| 业务用户   | 下单的业客用户                    |
| 订单原金额  | 支付给网点的金额（网点成本）             |
| 推荐客户价  | 基于30%利润率计算：`网点成本 × 1.4286` |
| 最终结算金额 | 财务人员确认的金额（默认=推荐客户价）        |
| 结算状态   | 待确认/已确认/已开票/已收款            |
| 创建时间   | 自动记录                       |

### 3.2 金额计算规则

**订单原金额**：

- 订单界面显示的支付给网点的金额（网点成本）
- 从订单记录中获取

**推荐客户价计算**：

```
推荐客户价 = 订单原金额 ÷ (1 - 30%)
           = 订单原金额 × 1.4286
```

**最终结算金额**：

- 默认值 = 推荐客户价
- 允许财务人员手动输入调整
- 调整后保存为最终结算金额

### 3.3 筛选功能

**按业务用户筛选**：

- 下拉选择业务用户
- 支持搜索功能
- 选择后显示该业务用户的所有结算记录

**按时间段筛选**：

- 日期范围选择器
- 支持快捷选择：今天/本周/本月/本年/自定义
- 筛选条件：创建时间在选定范围内

**筛选组合**：

- 支持业务用户 + 时间段组合筛选
- 筛选结果分页显示

### 3.4 一键开票功能

**开票流程**：

- 选择一条或多条结算记录
- 点击"一键开票"按钮
- 系统生成发票（PDF格式）
- 开票后更新结算状态为"已开票"

**发票内容**：

- 发票号（自动生成）
- 开票日期
- 购买方信息（业务用户）
- 销售方信息（红美物流）
- 结算明细（订单编号、金额）
- 税率（默认税率）

### 3.5 结算记录管理

**列表展示**：

| 列名     | 说明              |
| ------ | --------------- |
| 结算ID   | 自动编号            |
| 订单编号   | 关联订单            |
| 业务用户   | 客户名称            |
| 订单原金额  | 网点成本            |
| 推荐客户价  | 系统推荐            |
| 最终结算金额 | 确认金额            |
| 结算状态   | 待确认/已确认/已开票/已收款 |
| 创建时间   | 记录创建时间          |

**操作**：

- 查看详情
- 编辑最终结算金额
- 开票
- 标记收款

## 4. Acceptance Criteria

### 订单联动

- [ ] 订单回单拍照上传后（order\_status = completed），财务结算列表自动新增一条记录
- [ ] 结算记录正确关联订单编号和业务用户
- [ ] 结算记录正确记录订单原金额

### 金额计算

- [ ] 推荐客户价 = 订单原金额 × 1.4286
- [ ] 默认显示推荐客户价为最终结算金额
- [ ] 财务人员可以手动修改最终结算金额
- [ ] 修改后的金额被正确保存

### 筛选功能

- [ ] 支持按业务用户筛选结算记录
- [ ] 支持按时间段筛选结算记录
- [ ] 支持业务用户+时间段组合筛选
- [ ] 筛选结果分页展示

### 一键开票

- [ ] 选中结算记录后，可以一键生成发票
- [ ] 发票包含完整的结算明细
- [ ] 开票后结算状态更新为"已开票"

## 5. Edge Cases

- 同一订单已存在结算记录时，避免重复创建
- 订单金额为0时，推荐客户价也为0
- 业务用户被删除时，结算记录保留历史数据
- 开票失败时，显示错误提示，允许重试
