# 物流系统V4流程优化实施计划

## 项目概述

基于物流系统V4流程前后端设计评估报告，针对当前系统与V4流程不匹配的问题，制定详细的优化实施计划。

**评估结论**：
- 完整性：50%
- 逻辑性：60%
- 扩展性：40%
- 业务贴合度：55%

**优化目标**：使系统完整支持V4流程的13个任务和14个状态流转

---

### 阶段二：后端API开发（P0）

#### 任务 2.1：比价流程API

##### 2.1.1 派发比价请求API
- **优先级**: P0
- **依赖**: 任务 1.1
- **接口**: `POST /api/dispatch/request-quotes`
- **参数**:
  ```json
  {
    "orderId": 123,
    "networkIds": [1, 2, 3]
  }
  ```
- **逻辑**:
  1. 验证订单存在
  2. 验证网点列表
  3. 更新订单状态为"比价中"
  4. 生成比价请求记录
- **返回**:
  ```json
  {
    "code": 200,
    "message": "比价请求已发送",
    "data": {
      "orderId": 123,
      "quotes": [...]
    }
  }
  ```

##### 2.1.2 网点报价API
- **优先级**: P0
- **依赖**: 任务 2.1.1
- **接口**: `POST /api/network/quote`
- **参数**:
  ```json
  {
    "orderId": 123,
    "networkId": 1,
    "baseFee": 100.0,
    "finalPrice": 142.86,
    "transitDays": 2
  }
  ```
- **逻辑**:
  1. 验证网点权限
  2. 保存报价信息
  3. 更新订单比价状态
- **返回**:
  ```json
  {
    "code": 200,
    "message": "报价已提交",
    "data": { quoteId: 456 }
  }
  ```

##### 2.1.3 选择网点API（扩展）
- **优先级**: P0
- **依赖**: 任务 2.1.2
- **接口**: `POST /api/dispatch/select-quote`
- **扩展逻辑**:
  1. 选择最低价报价
  2. 更新订单selectedNetworkId
  3. 计算客户价（底价 × 1.4286）
  4. 更新订单pricingStatus为"待确认"
  5. 记录价格确认时间

##### 2.1.4 推送报价给客户API
- **优先级**: P1
- **依赖**: 任务 2.1.3
- **接口**: `POST /api/dispatch/push-quote`
- **参数**:
  ```json
  {
    "orderId": 123,
    "pushType": "sms|app|wechat"
  }
  ```
- **逻辑**:
  1. 生成报价推送记录
  2. 调用消息推送服务
  3. 更新推送状态

#### 任务 2.2：司机管理API

##### 2.2.1 分配提货司机API
- **优先级**: P0
- **依赖**: 任务 1.1
- **接口**: `POST /api/dispatch/assign-pickup-driver`
- **参数**:
  ```json
  {
    "orderId": 123,
    "driverId": 1
  }
  ```
- **逻辑**:
  1. 验证司机可用
  2. 更新订单pickupDriverId
  3. 更新订单状态为"待提货"
  4. 发送提货任务给司机
- **返回**:
  ```json
  {
    "code": 200,
    "message": "提货司机已分配",
    "data": { pickupTaskId: 789 }
  }
  ```

##### 2.2.2 分配配送司机API
- **优先级**: P0
- **依赖**: 任务 2.3
- **接口**: `POST /api/dispatch/assign-delivery-driver`
- **参数**:
  ```json
  {
    "orderId": 123,
    "driverId": 2
  }
  ```
- **逻辑**:
  1. 验证司机可用
  2. 更新订单deliveryDriverId
  3. 更新订单状态为"待配送"
  4. 发送配送任务给司机

#### 任务 2.3：网点操作API

##### 2.3.1 确认收货API
- **优先级**: P0
- **依赖**: 任务 2.2.1
- **接口**: `POST /api/network/confirm-receive`
- **参数**:
  ```json
  {
    "orderId": 123,
    "networkId": 1,
    "checkResult": "ok|exception",
    "remark": "货物完好"
  }
  ```
- **逻辑**:
  1. 验证订单当前状态
  2. 验证网点权限
  3. 更新warehouseStatus为"已入库"
  4. 记录warehouseConfirmTime
  5. 更新订单状态为"已入库"

##### 2.3.2 网点订单列表API
- **优先级**: P0
- **依赖**: 任务 2.3.1
- **接口**: `GET /api/network/orders`
- **参数**:
  - `networkId`: 网点ID
  - `status`: 订单状态（可选）
- **返回**: 网点相关的订单列表

#### 任务 2.4：状态更新API

##### 2.4.1 扩展状态更新API
- **优先级**: P0
- **依赖**: 任务 2.2
- **接口**: `POST /api/order/update-status-extended`
- **参数**:
  ```json
  {
    "orderId": 123,
    "status": 5,
    "statusType": "logistics|warehouse|pricing",
    "remark": "司机已取货",
    "location": "北京市朝阳区"
  }
  ```
- **逻辑**:
  1. 根据statusType更新对应状态
  2. 记录物流轨迹
  3. 更新tracking表

ALTER TABLE orders ADD COLUMN warehouse_status INT DEFAULT 0 COMMENT '0-待入库 1-已入库';
ALTER TABLE orders ADD COLUMN price_confirmed_time DATETIME;
ALTER TABLE orders ADD COLUMN picked_up_time DATETIME;
ALTER TABLE orders ADD COLUMN delivered_to_network_time DATETIME;
ALTER TABLE orders ADD COLUMN warehouse_confirm_time DATETIME;

-- 新增索引
CREATE INDEX idx_pickup_driver ON orders(pickup_driver_id);
CREATE INDEX idx_delivery_driver ON orders(delivery_driver_id);
CREATE INDEX idx_selected_network ON orders(selected_network_id);
