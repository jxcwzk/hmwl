# 财务结算界面功能 PRD

## Status
- [x] 已完成（2026-03-12）

## 1. Overview

- **需求名称**：财务结算界面功能增强
- **需求描述**：实现订单支付后自动生成结算记录、支持按业务用户和时间段筛选、提供一键开票功能
- **目标用户**：财务人员、运营管理人员
- **解决的问题**：
  1. 订单支付后需手动创建结算记录，工作量大且易遗漏
  2. 现有结算列表缺少筛选功能，查找困难
  3. 开票流程繁琐，需要简化

## 2. User Stories

- **Story 1**：作为财务人员，我希望订单完成支付后自动生成结算记录，这样我不需要手动创建，避免遗漏
- **Story 2**：作为财务人员，我希望能够按客户和时间段筛选结算记录，这样我可以快速找到需要的账单
- **Story 3**：作为财务人员，我希望能够一键生成发票，这样开票流程更高效

## 3. Functional Requirements

### 3.1 订单-结算联动
- 当订单状态变更为"已完成支付"时，自动创建一条结算记录
- 结算记录应包含：订单ID、订单编号、客户ID、金额、支付方式等
- 结算类型为"客户应收"(type=0)

### 3.2 结算记录筛选
- 支持按客户名称/ID筛选
- 支持按时间范围筛选（开始日期、结束日期）
- 支持按结算状态筛选（未结算、已结算、已付款）
- 支持多条件组合筛选

### 3.3 一键开票
- 在结算记录列表提供"开票"操作按钮
- 点击后生成发票记录
- 发票包含：发票号、客户信息、金额、日期等
- 发票状态：待开、已开

## 4. Acceptance Criteria

### 4.1 订单-结算联动
- [ ] 订单完成支付后，数据库settlement表新增一条记录
- [ ] 结算记录的orderId、amount、paymentMethod与订单一致
- [ ] 结算记录的type值为0（客户应收）
- [ ] 结算记录的status值为0（未结算）

### 4.2 结算记录筛选
- [ ] 筛选页面包含：客户选择器、日期范围选择器、状态选择器
- [ ] 选择筛选条件后，列表只显示匹配的记录
- [ ] 清空筛选条件后，显示所有记录
- [ ] 分页功能正常，每页显示10条

### 4.3 一键开票
- [ ] 每条结算记录行有"开票"按钮
- [ ] 点击开票按钮，弹出确认对话框
- [ ] 确认后，数据库invoice表新增一条记录
- [ ] 开票按钮点击后变为"已开票"状态，不可重复点击

## 5. Edge Cases

- **重复开票**：已开票的结算记录不允许再次开票
- **订单取消**：如果订单支付后取消，对应的结算记录状态应同步更新
- **批量筛选**：当筛选结果为空时，显示"暂无数据"提示

## 6. Priority

- **高优先级**
- 理由：财务结算是业务闭环的关键环节，直接影响运营效率
