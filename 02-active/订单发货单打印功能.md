## 1. Overview

**需求描述**：为每一条订单记录提供打印发货单功能，输出PDF格式文件，用作回单凭证

**目标用户**：物流公司操作人员、仓库人员

**解决的问题**：

- 客户需要物理回单凭证进行拍照上传
- 现有订单详情页包含报价信息，不适合直接打印

## 2. User Stories

| 角色   | 场景    | 期望                |
| ---- | ----- | ----------------- |
| 仓库人员 | 完成拣货后 | 可以打印该订单的发货单       |
| 仓库人员 | 打印发货单 | 发货单不包含价格信息，保护商业机密 |
| 客户   | 收到货物时 | 收到发货单作为签收凭证       |

## 3. Functional Requirements

### 3.1 打印入口

- 在订单列表页面，每行订单提供"打印发货单"按钮
- 在订单详情页面，提供"打印发货单"按钮

### 3.2 发货单内容

**固定发件人信息**：

| 字段    | 内容          |
| ----- | ----------- |
| 发件人姓名 | 郭红美         |
| 联系电话  | 15315162177 |
| 联系地址  | 靖江红美物流      |

**从订单获取的收件人信息**：

| 字段    | 内容         |
| ----- | ---------- |
| 收件人姓名 | 从订单收货人获取   |
| 联系电话  | 从订单收货人电话获取 |
| 收货地址  | 从订单收货地址获取  |

**订单信息**：

| 字段   | 内容         |
| ---- | ---------- |
| 订单编号 | 订单唯一标识     |
| 订单类型 | 整车/零担/快递   |
| 货物信息 | 品名         |
| 重量   | 实际重量(kg)   |
| 体积   | 体积(m³)     |
| 备注   | 订单备注       |
| 二维码  | 包含订单编号的二维码 |
| Logo | 红美物流标识     |

### 3.3 PDF输出要求

- 页面尺寸：A4
- 布局：简洁清晰，适合打印
- 不包含任何报价信息（单价、总价等）
- 二维码可用于扫描识别订单

### 3.4 技术实现

- 前端使用PDF生成库（如jsPDF、html2pdf）
- 直接在浏览器端生成PDF并触发打印

## 4. Acceptance Criteria

- [ ] 订单列表页面每行有"打印发货单"按钮
- [ ] 订单详情页面有"打印发货单"按钮
- [ ] 点击后生成PDF并打开打印预览
- [ ] 发货单包含所有必需信息（收件人、货物、重量、体积、备注、二维码）
- [ ] 发货单不包含任何价格信息
- [ ] 发件人信息显示固定内容（郭红美、15315162177、靖江红美物流）
- [ ] 二维码可以正常扫描识别订单

## 5. Edge Cases

- 订单缺少收件人信息时，打印时显示提示
- 订单备注为空时，该区域留空或显示"/"
- 网络不稳定时，PDF生成不受影响（前端子生成）
- 二维码生成失败时，可以不显示或显示订单编号代替

## 6. Priority

- **High** - 客户回单拍照上传是常见业务需求，关系到资金回笼
