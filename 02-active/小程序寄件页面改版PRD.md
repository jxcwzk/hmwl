## 1. Overview

- **需求描述**：修改小程序前端寄件页面，与网页端元素保持一致，针对客户角色简化操作流程
- **目标用户**：小程序客户（寄件人）
- **解决的问题**：小程序寄件功能不完善，缺少货物照片上传，与网页端功能不一致

## 2. User Stories

| 角色 | 场景 | 需求 |
|------|------|------|
| 客户 | 选择发件人 | 我希望可以选择已有的发件人信息，也可以手动输入新的发件人信息 |
| 客户 | 选择收件人 | 我希望可以选择已有的收件人信息，也可以手动输入新的收件人信息 |
| 客户 | 填写货物信息 | 我希望能够填写货物描述、重量、体积和备注 |
| 客户 | 上传货物照片 | 我希望能够拍照或从相册选择货物照片上传 |
| 客户 | 提交订单 | 我希望能够一键提交订单，订单创建成功后自动上传照片 |

## 3. Functional Requirements

### 3.1 发件人信息模块
- [ ] 展示发件人选择器，下拉列表显示已有发件人（从 business-customer 接口获取，type=0）
- [ ] 选择发件人后自动填充姓名、电话、地址
- [ ] 支持手动输入发件人信息（姓名、电话、地址）
- [ ] 选择已有发件人时，手动输入框显示为只读或清空

### 3.2 收件人信息模块
- [ ] 展示收件人选择器，下拉列表显示已有收件人（从 business-customer 接口获取，type=1）
- [ ] 选择收件人后自动填充姓名、电话、地址
- [ ] 支持手动输入收件人信息（姓名、电话、地址）
- [ ] 选择已有收件人时，手动输入框显示为只读或清空

### 3.3 货物信息模块
- [ ] 描述字段（必填）
- [ ] 重量字段，单位 kg（必填）
- [ ] 体积字段，单位 m³（选填）
- [ ] 备注字段（选填）

### 3.4 货物照片上传模块
- [ ] 展示拍照/相册选择按钮
- [ ] 支持多选，最多5张照片
- [ ] 照片预览功能
- [ ] 删除已选照片功能
- [ ] 调用 `/order/goods-image/upload/cos` 接口上传

### 3.5 订单提交模块
- [ ] 表单验证（发件人信息、收件人信息、货物描述和重量必填）
- [ ] 调用 `/order` 接口创建订单
- [ ] 订单创建成功后上传货物照片
- [ ] 提交过程中显示 loading 状态
- [ ] 成功后跳转或提示

### 3.6 页面调整
- [ ] 移除"寄件方式"选择（上门取件/网点寄件）
- [ ] 移除"服务类型"选择（标准快递/加急/冷链）
- [ ] 移除"费用估算"展示（简化客户操作）

## 4. Acceptance Criteria

### 4.1 发件人/收件人选择
- [ ] 能够正确获取并展示已有发件人/收件人列表
- [ ] 选择后正确填充到对应输入框
- [ ] 手动输入时能够正常编辑

### 4.2 货物信息
- [ ] 所有字段能够正常输入
- [ ] 描述和重量为必填，不填写无法提交

### 4.3 照片上传
- [ ] 能够正常调用相机拍照
- [ ] 能够从相册选择照片
- [ ] 最多选择5张，超过提示用户
- [ ] 能够删除已选照片
- [ ] 能够预览照片
- [ ] 照片能够成功上传到服务器

### 4.4 订单提交
- [ ] 验证所有必填项
- [ ] 订单创建成功
- [ ] 照片上传成功
- [ ] 用户看到成功提示

## 5. Edge Cases

| 场景 | 处理方式 |
|------|----------|
| 未登录 | 提示用户先登录 |
| 发件人/收件人列表为空 | 允许手动输入 |
| 网络请求失败 | 显示错误提示，允许重试 |
| 照片上传部分失败 | 提示部分照片上传失败 |
| 超过最大照片数量 | 提示已达到最大数量限制 |

## 9. Technical Notes

- API_BASE 地址需要根据实际部署环境配置
- 使用 wx.getStorageSync('token') 和 wx.getStorageSync('businessUserId') 获取认证信息
- 接口列表：
  - GET `/business-customer/listByBusinessUserIdAndType/{businessUserId}/0` - 获取发件人列表
  - GET `/business-customer/listByBusinessUserIdAndType/{businessUserId}/1` - 获取收件人列表
  - POST `/order` - 创建订单
  - POST `/order/goods-image/upload/cos` - 上传货物照片
