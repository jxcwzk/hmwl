## 1. Overview

**需求描述**：在前端订单创建页面设置必填项，并基于业务用户筛选发件人和收件人信息

**目标用户**：物流公司操作人员、订单录入员

**解决的问题**：

- 防止重要物流信息遗漏
- 减少发件人/收件人选择错误
- 提高订单数据完整性

## 2. User Stories

| 角色    | 场景      | 期望                           |
| ----- | ------- | ---------------------------- |
| 订单录入员 | 创建订单时   | 必须填写业务用户、发件人、收件人等关键信息，否则无法提交 |
| 订单录入员 | 选择业务用户后 | 发件人下拉列表只显示该业务用户关联的发件人        |
| 订单录入员 | 选择业务用户后 | 收件人下拉列表只显示该业务用户关联的收件人        |
| 系统管理员 | 配置订单字段  | 可以设置哪些字段为必填项                 |

## 3. Functional Requirements

### 3.1 必填项设置

- 业务用户（customer）：必填
- 发件人姓名：必填
- 发件人电话：必填
- 发件人地址：必填
- 收件人姓名：必填
- 收件人电话：必填
- 收件人地址：必填
- 货物信息（品名、数量）：必填
- 重量：必填
- 前端表单提交时校验必填项，未填写则提示用户

### 3.2 发件人/收件人筛选

- 选择业务用户后，自动过滤发件人列表，只显示该业务用户关联的发件人
- 选择业务用户后，自动过滤收件人列表，只显示该业务用户关联的收件人
- 支持搜索功能，可在下拉框中搜索发件人/收件人姓名或电话

### 3.3 交互流程优化

- 页面加载时，先选择业务用户
- 选择业务用户后，再选择发件人和收件人
- 发件人/收件人信息可手动新增或从关联列表选择

## 4. Acceptance Criteria

- [ ] 订单创建页面，业务用户字段为必填，未选择无法提交
- [ ] 订单创建页面，发件人相关字段（姓名、电话、地址）为必填
- [ ] 订单创建页面，收件人相关字段（姓名、电话、地址）为必填
- [ ] 选择业务用户后，发件人下拉列表只显示该业务用户关联的发件人
- [ ] 选择业务用户后，收件人下拉列表只显示该业务用户关联的收件人
- [ ] 未填写必填项时，页面显示清晰的错误提示
- [ ] 必填项校验在客户端（前端）完成

## 5. Edge Cases

- 业务用户尚未关联任何发件人/收件人时，下拉列表显示为空，需提示用户先添加
- 发件人或收件人信息变更时，需要及时更新关联关系
- 兼容老数据，业务用户字段可以为空（兼容历史订单）
