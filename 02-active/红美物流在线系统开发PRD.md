## 1. 图片上传大小限制与压缩功能 PRD

## Status
- [x] 待用户确认
- [x] 已确认，待开发
- [x] 开发中
- [x] 已完成（2026-03-12）

## 1. Overview

- **需求描述**：在图片上传到腾讯云COS前进行大小判断，大于2MB的图片自动压缩后再上传，控制CDN流量费用
- **目标用户**：运营人员、财务人员
- **解决的问题**：解决腾讯云COS图片预览产生的CDN流量费用过高问题

## 2. User Stories

- **Story 1**: 作为运营人员，我希望上传小于2MB的图片时直接上传，这样加载速度快
- **Story 2**: 作为运营人员，我希望上传大于等于2MB的图片时自动压缩，这样不影响CDN费用
- **Story 3**: 作为财务人员，我希望上传的发货单/回单图片体积适中，这样查看方便且节省成本

## 3. Functional Requirements

- **FR-001**: 图片大小判断 - 上传图片前判断文件大小
- **FR-002**: 小图直传 - 小于2MB的图片直接上传到COS
- **FR-003**: 大图压缩 - 大于等于2MB的图片自动压缩至小于2MB后再上传
- **FR-004**: 质量控制 - 压缩质量保持70%
- **FR-005**: 格式支持 - 支持 jpg、png、gif、bmp、webp 格式
- **FR-006**: 服务覆盖 - 发货单图片和回单图片上传均应用此规则

## 4. Acceptance Criteria

- **AC-001**: 上传1.5MB的图片，文件直接上传，不进行压缩处理
- **AC-002**: 上传5MB的jpg图片，压缩后文件小于2MB，上传成功
- **AC-003**: 上传3MB的png图片，压缩后文件小于2MB，上传成功
- **AC-004**: 上传10MB的图片，经过多次压缩后文件小于2MB，上传成功
- **AC-005**: 压缩后的图片仍然清晰可读，70%质量肉眼观察无明显失真

## 5. Edge Cases

- **EC-001**: 图片已经是极小尺寸（如100KB），直接上传
- **EC-002**: 压缩后仍大于2MB（极少见），继续降低质量直到可接受或达到最低质量阈值
- **EC-003**: 不支持的图片格式，应返回错误提示

## 6. Priority

- **High** - 直接影响CDN流量费用控制

## 7. Timeline

- 已完成实现

## 2. 多角色订单管理系统 PRD

## Status
- [ ] 待用户确认
- [x] 已确认，待开发
- [ ] 开发中
- [ ] 已完成

## 1. Overview

- **需求描述**：实现多角色订单管理系统，包括管理员、司机、客户和新增的网点角色，每个角色拥有不同的权限和功能。
- **目标用户**：系统管理员、司机、业务客户、物流网点。
- **解决的问题**：优化订单管理流程，提高各角色的工作效率，增强系统的可扩展性和用户体验。

## 2. User Stories

- **Story 1**：作为管理员，我希望查看所有订单，无需自动筛选，以便全面了解系统中的订单情况。
- **Story 2**：作为管理员，我希望在订单详情页面指派订单给司机，以便司机能够接收并处理相关订单。
- **Story 3**：作为管理员，我希望有估价和指派网点的功能，以便为订单提供合理的价格估算和物流路径。
- **Story 4**：作为管理员，我希望获得网点提供的底价，乘以系数作为给业务用户的报价，并可以手动修改，以便灵活调整价格策略。
- **Story 5**：作为司机，我希望看到指派给自己的订单，以便了解自己需要处理的任务。
- **Story 6**：作为客户，我希望看到自己的订单，以便跟踪订单状态。
- **Story 7**：作为客户，我希望看到物流进展，以便了解订单的实时状态。
- **Story 8**：作为客户，我希望看到预估价，以便了解订单的预计成本。
- **Story 9**：作为网点，我希望能够更新物流进度，以便客户能够实时了解订单状态。
- **Story 10**：作为网点，我希望对于订单价格做最终确认，以便确保价格的准确性。
- **Story 11**：作为网点，我希望提供报价给红美物流，以便系统能够生成合理的客户报价。

## 3. Functional Requirements

### 3.1 管理员功能
- 查看所有订单，无需自动筛选
- 订单详情页面支持指派订单给司机
- 估价功能，基于网点报价和系数计算客户报价
- 指派网点功能
- 手动修改客户报价

### 3.2 司机功能
- 查看指派给自己的订单

### 3.3 客户功能
- 查看自己的订单
- 查看物流进展
- 查看预估价

### 3.4 网点功能
- 更新物流进度
- 最终确认订单价格
- 提供报价给红美物流

## 4. Acceptance Criteria

### 4.1 管理员功能
- 管理员登录后能看到所有订单列表
- 点击订单详情后能看到指派司机的选项
- 能看到网点提供的底价和计算后的客户报价
- 能手动修改客户报价
- 能指派网点给订单

### 4.2 司机功能
- 司机登录后能看到指派给自己的订单列表
- 订单列表显示订单的基本信息

### 4.3 客户功能
- 客户登录后能看到自己的订单列表
- 点击订单详情能看到物流进展
- 能看到订单的预估价

### 4.4 网点功能
- 网点登录后能看到分配给自己的订单
- 能更新订单的物流进度
- 能提供订单的报价
- 能最终确认订单价格

## 5. Edge Cases

- **5.1 订单状态变更**：当订单状态发生变化时，确保各角色能及时看到更新。
- **5.2 价格计算**：当网点报价为0或负数时的处理。
- **5.3 权限控制**：确保各角色只能访问自己权限范围内的功能。
- **5.4 网络异常**：当网络异常时，确保数据的一致性和可靠性。

## 6. Priority

- **优先级**：高
- **理由**：涉及核心业务流程，影响多个角色的使用体验，是系统的重要功能。

## 7. Timeline

- **预估时间**：4周
  - 需求分析和设计：1周
  - 后端开发：1.5周
  - 前端开发：1周
  - 测试和调整：0.5周

## 3. 小程序后端API对接与登录系统 PRD

## Status
- [x] 待用户确认
- [x] 已确认，待开发
- [x] 开发中
- [x] 已完成

## 完成时间
- 2026-03-13

## 1. Overview

### 需求描述
将微信小程序从微信云开发迁移到调用后端Java API，实现小程序、VUE前端、后端云服务三者数据互通，同时保留云开发部分功能（图片上传）。优先对接订单模块，并实现微信自动登录功能。

### 目标用户
- 小程序终端用户（客户、司机）
- 红美物流后台管理员

### 解决的问题
- 小程序数据与Vue前端不一致的问题
- 手动管理用户身份的繁琐
- 三端数据统一，方便后续维护

### 2.2 订单模块

|-------|------|

| Story | 描述 |
|-------|------|
| US7 | 作为开发人员，小程序能在开发阶段连接本地后端，发布后连接云端数据库 |
| US8 | 作为开发人员，图片上传保留云开发功能 |

---

## 3. Functional Requirements

### 3.1 登录与身份管理

#### FR1: 微信自动登录
- 小程序调用wx.login获取code
- 将code发送到后端Java API换取openid
- 后端根据openid查找或创建用户记录
- 返回用户信息（包含身份状态）

#### FR2: 身份状态查询
- 用户登录后，前端查询用户身份状态
- 状态包括：未分配、待审核、已分配（客户/司机/管理员）
- 未分配身份时，提示用户联系管理员

#### FR3: 后台用户管理（VUE端）
- 管理员查看所有小程序注册用户
- 管理员为用户指派身份角色
- 支持角色：客户、司机、网点管理员、系统管理员

#### FR4: 小程序端身份展示
- 用户登录后显示个人中心
- 展示当前身份状态
- 未分配身份时，显示提示信息

### 3.2 订单模块对接

#### FR5: 订单列表API对接
- 小程序调用后端 `/api/order/list` 获取订单列表
- 实现分页查询
- 对接现有订单筛选功能

#### FR6: 订单详情API对接
- 小程序调用后端 `/api/order/{id}` 获取详情
- 展示订单完整信息

#### FR7: 创建订单API对接
- 小程序调用后端 `/api/order/add` 创建订单
- 支持订单相关图片上传

### 3.3 其他模块对接（可选扩展）

#### FR8: 司机列表对接
- 小程序调用后端 `/api/driver/list` 获取司机列表

#### FR9: 网点列表对接
- 小程序调用后端 `/api/network/list` 获取网点列表

#### FR10: 车辆列表对接
- 小程序调用后端 `/api/vehicle/list` 获取车辆列表

### 3.4 基础设施

#### FR11: 环境配置
- 开发环境：baseURL = `http://localhost:8081`
- 生产环境：baseURL = 云端服务器地址
- 通过配置文件切换

#### FR12: 云函数保留
- 保留图片上传相关云函数
- 图片存储继续使用腾讯云COS

## 1. Overview

## 1. Overview

**需求描述**：根据发货地址和收货地址自动计算两点之间的距离，无需手动填写

**目标用户**：物流公司操作人员、订单录入员

**解决的问题**：

- 手动填写距离容易出错
- 距离是运费计算的关键参数
- 自动化提升工作效率

## 2. User Stories

| 角色    | 场景    | 期望                       |
| ----- | ----- | ------------------------ |
| 订单录入员 | 创建订单时 | 输入发货地址和收货地址后，系统自动计算距离并填充 |
| 订单录入员 | 编辑订单时 | 如果修改了地址，距离自动重新计算         |
| 财务人员  | 查看订单时 | 距离数据准确，可用于运费核算           |

## 3. Functional Requirements

### 3.1 距离计算触发时机

- 订单创建时：当发货地址和收货地址都有值时自动计算
- 订单编辑时：当地址发生变化时自动重新计算
- 支持手动触发重新计算

### 3.2 地址来源

- 发货地址：来源于选择的发货网点（网点地址）
- 收货地址：来源于选择的收货网点（网点地址）
- 或者直接输入详细地址

### 3.3 计算方案

- 优先使用地图API计算实际道路距离
- 备选方案：计算两点间的直线距离（使用经纬度）
- 推荐地图服务：腾讯地图（已有配置）或高德地图

### 3.4 距离数据使用

- 距离单位：公里(km)
- 存储到订单记录的距离字段
- 参与运费计算：运费 = 距离 × 单价

### 3.5 异常处理

- 地址解析失败时，显示提示信息，允许手动输入
- 地图API调用失败时，使用备用直线距离算法
- 无效地址时，提示用户检查地址信息

## 4. Acceptance Criteria

- [ ] 订单创建页面，当发货地址和收货地址都有值时自动计算距离
- [ ] 距离计算完成后自动填充到距离字段
- [ ] 编辑订单时，修改地址后距离自动重新计算
- [ ] 距离单位显示为公里(km)
- [ ] 地址解析失败时，允许手动输入距离
- [ ] 距离数据参与运费计算

## 5. Edge Cases

- 发货地址或收货地址为空时，不触发计算
- 相同的发货和收货地址，距离显示为0
- 地图API返回异常时的降级处理
- 历史订单的距离数据不受影响

## 8. Technical Notes

### 距离计算公式

**直线距离（备用）**：

```
距离(km) = √[(x2-x1)² + (y2-y1)²] × 111.32
```

其中111.32是地球每度约等于的公里数

**地图API推荐参数**：

- 腾讯地图：WebService API - 距离计算
- 返回格式：JSON
- 单位：米（转换为公里）

| 财务人员 | 查看结算记录时 | 可以按业务用户筛选，查看该客户的所有结算记录 |
| 财务人员 | 查看结算记录时 | 可以按时间段筛选，查看特定期间的结算记录   |
| 财务人员 | 录入结算金额时 | 系统给出基于30%利润率的推荐客户报价    |
| 财务人员 | 录入结算金额时 | 可以手动调整最终结算金额           |
| 财务人员 | 开票时     | 可以一键生成发票               |

## 3. Functional Requirements

### 3.1 订单联动（结算记录自动生成）

**触发条件：订单完成（回单拍照上传）**

- 监听订单状态变化
- 当 order\_status = completed（回单拍照上传）时，自动创建结算记录
- 记录订单编号、业务用户、订单金额（网点成本）
- 不在订单支付完成时生成结算记录

**结算记录内容**：

| 字段     | 说明                         |
| ------ | -------------------------- |
| 结算ID   | 自动生成，唯一标识                  |
| 订单编号   | 关联的订单编号                    |
| 业务用户   | 下单的业客用户                    |
| 订单原金额  | 支付给网点的金额（网点成本）             |
| 推荐客户价  | 基于30%利润率计算：`网点成本 × 1.4286` |
| 最终结算金额 | 财务人员确认的金额（默认=推荐客户价）        |
| 结算状态   | 待确认/已确认/已开票/已收款            |
| 创建时间   | 自动记录                       |

### 3.2 金额计算规则

**订单原金额**：

- 订单界面显示的支付给网点的金额（网点成本）
- 从订单记录中获取

**推荐客户价计算**：

```
推荐客户价 = 订单原金额 ÷ (1 - 30%)
           = 订单原金额 × 1.4286
```

**最终结算金额**：

- 默认值 = 推荐客户价
- 允许财务人员手动输入调整
- 调整后保存为最终结算金额

### 3.3 筛选功能

**按业务用户筛选**：

- 下拉选择业务用户
- 支持搜索功能
- 选择后显示该业务用户的所有结算记录

**按时间段筛选**：

- 日期范围选择器
- 支持快捷选择：今天/本周/本月/本年/自定义
- 筛选条件：创建时间在选定范围内

**筛选组合**：

- 支持业务用户 + 时间段组合筛选
- 筛选结果分页显示

### 3.4 一键开票功能

**开票流程**：

- 选择一条或多条结算记录
- 点击"一键开票"按钮
- 系统生成发票（PDF格式）
- 开票后更新结算状态为"已开票"

**发票内容**：

- 发票号（自动生成）
- 开票日期
- 购买方信息（业务用户）
- 销售方信息（红美物流）
- 结算明细（订单编号、金额）
- 税率（默认税率）

### 3.5 结算记录管理

**列表展示**：

| 列名     | 说明              |
| ------ | --------------- |
| 结算ID   | 自动编号            |
| 订单编号   | 关联订单            |
| 业务用户   | 客户名称            |
| 订单原金额  | 网点成本            |
| 推荐客户价  | 系统推荐            |
| 最终结算金额 | 确认金额            |
| 结算状态   | 待确认/已确认/已开票/已收款 |
| 创建时间   | 记录创建时间          |

**操作**：

- 查看详情
- 编辑最终结算金额
- 开票
- 标记收款
