# 测试用例文档

## API测试用例

### 业务用户接口测试

#### 1. 获取业务用户列表
- **测试用例ID**: API-USER-001
- **测试名称**: 获取业务用户列表
- **测试描述**: 测试获取所有业务用户列表的接口
- **测试步骤**: 
  1. 发送GET请求到 `/api/business-user/list`
  2. 验证响应状态码为200
  3. 验证响应数据为业务用户列表
- **预期结果**: 成功获取业务用户列表

#### 2. 根据业务用户ID获取业务客户列表
- **测试用例ID**: API-CUSTOMER-001
- **测试名称**: 根据业务用户ID获取业务客户列表
- **测试描述**: 测试根据业务用户ID获取业务客户列表的接口
- **测试步骤**: 
  1. 发送GET请求到 `/api/business-customer/listByBusinessUserId/{businessUserId}`
  2. 验证响应状态码为200
  3. 验证响应数据为指定业务用户的业务客户列表
- **预期结果**: 成功获取指定业务用户的业务客户列表

#### 3. 根据业务用户ID和类型获取业务客户列表
- **测试用例ID**: API-CUSTOMER-002
- **测试名称**: 根据业务用户ID和类型获取业务客户列表
- **测试描述**: 测试根据业务用户ID和类型获取业务客户列表的接口
- **测试步骤**: 
  1. 发送GET请求到 `/api/business-customer/listByBusinessUserIdAndType/{businessUserId}/0` 获取发件人信息
  2. 验证响应状态码为200
  3. 验证响应数据为指定业务用户的发件人信息列表
  4. 发送GET请求到 `/api/business-customer/listByBusinessUserIdAndType/{businessUserId}/1` 获取收件人信息
  5. 验证响应状态码为200
  6. 验证响应数据为指定业务用户的收件人信息列表
- **预期结果**: 成功获取指定业务用户的发件人和收件人信息列表

#### 4. 保存订单信息
- **测试用例ID**: API-ORDER-001
- **测试名称**: 保存订单信息
- **测试描述**: 测试保存订单信息的接口
- **测试步骤**: 
  1. 准备订单数据，包括发件人和收件人信息
  2. 发送POST请求到 `/api/order`
  3. 验证响应状态码为200
  4. 验证响应数据为true
- **预期结果**: 成功保存订单信息

## 性能测试用例

### API性能测试

#### 1. 并发请求测试
- **测试用例ID**: PERF-API-001
- **测试名称**: 并发请求测试
- **测试描述**: 测试API在并发请求下的性能
- **测试步骤**: 
  1. 发送50个并发请求到 `/api/business-user/list`
  2. 发送50个并发请求到 `/api/business-customer/list`
  3. 发送50个并发请求到 `/api/order/list`
- **预期结果**: 所有请求都能成功响应，平均响应时间不超过500ms

#### 2. 持续负载测试
- **测试用例ID**: PERF-API-002
- **测试名称**: 持续负载测试
- **测试描述**: 测试API在持续负载下的性能
- **测试步骤**: 
  1. 持续发送请求到 `/api/business-customer/list` 60秒
  2. 记录响应时间和错误率
- **预期结果**: 平均响应时间不超过500ms，错误率不超过1%
