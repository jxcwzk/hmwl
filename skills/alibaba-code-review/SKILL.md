# 阿里巴巴编码规则代码审查技能

## 1. 技能概述

本技能基于《阿里巴巴Java开发手册》，提供全面的代码审查功能，帮助开发团队确保代码质量和一致性。

## 2. 审查范围

- **代码风格**：缩进、换行、空格等
- **命名规范**：类名、方法名、变量名、常量名等
- **异常处理**：异常捕获、抛出、日志记录等
- **注释规范**：类注释、方法注释、代码注释等
- **安全问题**：SQL注入、XSS攻击、密码存储等
- **性能问题**：循环优化、内存使用、IO操作等
- **并发问题**：线程安全、锁使用等

## 3. 审查流程

1. **准备阶段**：收集待审查的代码文件
2. **规则检查**：按照阿里巴巴编码规范检查代码
3. **问题识别**：标记不符合规范的代码
4. **问题分类**：按照严重程度和类型分类问题
5. **生成报告**：生成详细的代码审查报告
6. **提供建议**：为每个问题提供改进建议

## 4. 核心规则

### 4.1 命名规范
- **类名**：使用大驼峰命名法，如 `UserService`
- **方法名**：使用小驼峰命名法，如 `getUserById`
- **变量名**：使用小驼峰命名法，如 `userName`
- **常量名**：使用全大写字母和下划线，如 `MAX_COUNT`
- **包名**：使用小写字母，如 `com.alibaba.service`

### 4.2 代码风格
- **缩进**：使用4个空格，不使用制表符
- **换行**：每行不超过120个字符
- **空格**：操作符前后、逗号后、括号内等位置添加空格
- **大括号**：左大括号不换行，右大括号单独换行

### 4.3 异常处理
- **捕获异常**：不要捕获 `Exception`，要捕获具体的异常类型
- **异常处理**：捕获异常后要进行适当处理，不能直接忽略
- **日志记录**：异常发生时要记录日志，包含足够的上下文信息
- **异常抛出**：向上层抛出异常时要添加适当的信息

### 4.4 注释规范
- **类注释**：包含类的功能描述、作者、创建日期等
- **方法注释**：包含方法的功能、参数、返回值、异常等
- **代码注释**：复杂代码要添加注释，说明实现思路
- **TODO注释**：使用 `// TODO:` 标记待完成的工作

### 4.5 安全规范
- **SQL注入**：使用参数化查询，避免拼接SQL语句
- **XSS攻击**：对用户输入进行过滤和转义
- **密码存储**：使用加盐哈希存储密码，不存储明文密码
- **权限控制**：实现严格的权限检查，防止越权访问

- **report_generator.py**：生成代码审查报告

### 5.2 参考资料

## 6. 使用方法

### 6.1 基本使用
1. 收集待审查的代码文件
2. 运行代码检查脚本
3. 生成审查报告
4. 分析报告并进行代码改进

### 6.2 示例
```bash
# 检查单个文件
python scripts/code_check.py --file src/main/java/com/hmwl/controller/RouteController.java

# 检查整个目录
python scripts/code_check.py --directory src/main/java

# 生成审查报告
python scripts/report_generator.py --input results.json --output report.html
```

## 7. 常见问题及解决方案

### 7.1 命名规范问题
- **问题**：变量名使用缩写或不清晰的命名
- **解决方案**：使用有意义的变量名，避免使用缩写

### 7.2 异常处理问题
- **问题**：捕获异常后直接忽略
- **解决方案**：记录日志或进行适当处理

### 7.3 代码风格问题
- **问题**：代码缩进不一致
- **解决方案**：使用IDE的自动格式化功能

### 7.4 安全问题
- **问题**：SQL语句拼接
- **解决方案**：使用参数化查询

## 8. 最佳实践

- **团队协作**：建立统一的编码规范，确保团队成员都遵循
- **持续集成**：在CI/CD流程中集成代码审查
- **定期培训**：定期组织编码规范培训，提高团队代码质量
- **工具辅助**：使用IDE插件和代码分析工具辅助代码审查
