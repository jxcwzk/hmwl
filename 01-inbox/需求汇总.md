## 1. 图片上传与存储

**文件**: `图片上传大小限制与压缩功能.md`

**状态**: ✅ 已完成 (2026-03-12)

**需求描述**:
- 在图片上传到腾讯云COS之前进行大小判断
- 图片小于2MB：直接上传原图
- 图片大于等于2MB：自动压缩至小于2MB后再上传
- 压缩质量：70%
- 支持多种图片格式（jpg、png、gif、bmp、webp）

**技术实现**:
- 使用 Thumbnailator 库进行图片压缩
- 修改 SenderImageServiceImpl 和 ReceiptImageServiceImpl

## 2. 多角色订单管理

**文件**: `多角色订单管理系统需求.md`

**状态**: ✅ 已完成

**需求描述**:
- **管理员界面**：
  - 查看所有订单，无需自动筛选
  - 点击订单详情时可以指派订单给司机
  - 有估价和指派网点的功能
  - 获得网点提供的底价，乘以系数作为给业务用户的报价，可手动修改
- **司机界面**：
  - 看到指派给自己的订单
- **客户界面**：
  - 看到自己的订单
  - 看到物流进展
  - 看到预估价
- **网点角色**：
  - 能够更新物流进度
  - 对于订单价格做最终确认
  - 提供报价给红美物流

## 3. 小程序/VUE/后端打通

**文件**: `小程序Vue后端三者打通需求.md`

**状态**: ✅ 已完成 (2026-03-13)

**需求描述**:
- 小程序改为调用后端Java API（和Vue前端一致）
- 三端数据源统一
- 保留云开发功能（图片上传等）
- 后续数据迁移到腾讯云数据库

**技术方案**:
- API调用: 从wx.cloud改为wx.request请求后端Java API
- 登录机制: 小程序端微信自动登录，登录后由红美物流后台管理员指派身份信息
