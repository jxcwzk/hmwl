# 线路管理与网点管理优化

## 原始想法
> 帮我修改下线路管理和网点管理界面，这两个页面应该是有关系的，一个线路有若干个网点，当获得确切的客户物流起止点信息后，匹配最合适的线路，基于确定的线路，选择合适的网点派发报价

## 详细描述

### 当前问题
- 线路(Route)和网点(NetworkPoint)当前是一对多关系（一个网点可以有多个线路）
- 缺少：一条线路经过多个网点的能力
- 调度派发时无法基于线路匹配网点

### 优化目标
1. **线路管理**：一条线路（如上海→北京）可以关联多个网点
2. **网点管理**：显示每个网点所在的线路
3. **调度派发**：
   - 根据客户订单的起止城市自动匹配线路
   - 基于匹配到的线路，展示沿线网点
   - 选择合适网点派发报价

### 业务流程
```
客户下单(发货地→收货地)
    ↓
系统匹配线路(起点城市→终点城市)
    ↓
显示沿线网点列表
    ↓
调度选择网点派发报价
    ↓
网点报价 → 调度确认 → 完成比价流程
```

## 数据结构设计

### 方案：线路-网点关联表

新建 `route_network_point` 关联表：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键 |
| route_id | Long | 线路ID |
| network_point_id | Long | 网点ID |
| sequence | Integer | 在线路上的顺序 |

### 修改现有表

**Route.java**:
- 移除 `networkPointId` (一对多改为多对多)
- 添加 `@ManyToMany` 或使用关联表

## 页面修改

### 1. 线路管理页面 (Route.vue)
- 显示每条线路的基本信息
- 展示该线路关联的网点列表
- 支持添加/移除网点
- 支持按起止城市搜索

### 2. 网点管理页面 (NetworkPoint.vue)
- 显示网点基本信息
- 展示该网点所在的线路列表

### 3. 调度管理页面 (Dispatch.vue)
- 选择网点派发时：
  - 显示起止城市
  - 自动匹配可用线路
  - 展示沿线网点供选择

## 优先级评估
- **高优先级**
- 这是调度核心流程的基础
