# 订单全生命周期操作时间记录

## 原始想法
> 现在红美物流有四个角色（客户、调度、网点、司机），并且有相应的流程，现在需要把每个角色在操作时候的时间记录下来，作为订单流的一部分，流程参照订单全生命周期

## 详细描述

### 四角色定义
- **客户**：下单、查看物流、确认签收
- **调度**：派发比价、安排提货、分配配送
- **网点**：报价、确认收货、物流更新
- **司机**：提货、配送、状态更新

### 订单全生命周期（V4流程状态机）
```
订单创建 → 派发比价 → 网点报价 → 选择报价 → 推送客户
    ↓
客户确认 → 安排提货 → 司机提货 → 送达网点 → 网点确认
    ↓
分配配送 → 配送中 → 客户签收 → 订单完成
```

### 需要记录的操作时间点
| 阶段 | 操作角色 | 记录时间点 |
|-----|---------|-----------|
| 订单创建 | 客户 | 下单时间 |
| 派发比价 | 调度 | 派发时间 |
| 网点报价 | 网点 | 报价时间 |
| 选择报价 | 客户 | 确认时间 |
| 安排提货 | 调度 | 安排时间 |
| 司机提货 | 司机 | 提货时间 |
| 送达网点 | 司机 | 送达时间 |
| 网点确认 | 网点 | 确认时间 |
| 分配配送 | 调度 | 分配时间 |
| 配送中 | 司机 | 出发时间 |
| 客户签收 | 客户 | 签收时间 |
| 订单完成 | 系统 | 完成时间 |

## 相关想法
- 可用于订单追溯和物流追踪展示给客户
- 可用于流程效率分析和超时预警
- 作为订单表扩展字段或单独的时间记录表
