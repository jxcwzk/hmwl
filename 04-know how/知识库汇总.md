## 1. 经验总结

**文件**: `红美物流在线系统开发经验总结.md`

**内容**:

### 项目概述
红美物流在线系统是一个多角色的物流管理系统，实现了从客户下单到订单完成的全流程管理。系统支持四个角色：客户、管理员、司机和网点，覆盖了物流业务的各个环节。

### 系统架构
```
前端层: Vue管理端 / 微信小程序 / 企业微信
    ↓
后端层: Spring Boot API / 业务逻辑服务 / 数据访问层
    ↓
数据层: MySQL数据库 / 腾讯云COS / Redis缓存
```

### 四角色系统
- **客户**: 下单、查看物流、确认签收
- **管理员**: 订单管理、调度指派、价格管理
- **司机**: 提货、配送、状态更新
- **网点**: 报价、确认收货、物流更新

### 关键经验
1. 多角色权限设计
2. 物流状态机实现
3. 图片压缩与COS存储
4. 微信小程序与后端API对接

## 2. 技术架构

**文件**: `红美物流在线系统技术架构设计.md`

**内容**:

### 技术栈
- **后端**: Spring Boot + MyBatis-Plus
- **前端**: Vue.js + 微信小程序
- **数据库**: MySQL
- **缓存**: Redis
- **存储**: 腾讯云COS
- **地图**: 腾讯地图API

### 数据库设计
- Order表: 订单主表
- Driver表: 司机表
- NetworkPoint表: 网点表
- BusinessCustomer表: 业务客户表

### API设计
- RESTful风格
- 统一响应格式
- 权限控制

### V4流程状态机
```
订单创建 → 派发比价 → 网点报价 → 选择报价 → 推送客户
    ↓
客户确认 → 安排提货 → 司机提货 → 送达网点 → 网点确认
    ↓
分配配送 → 配送中 → 客户签收 → 订单完成
```
