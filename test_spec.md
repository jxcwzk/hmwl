# 前后端测试规范文档

## 1. 测试目标
- 验证前后端链接是否正常
- 测试业务客户选择功能是否正常工作
- 验证订单界面的收件人信息是否正确显示
- 确保字符编码问题已解决

## 2. 测试环境
- 后端：Spring Boot 2.7.15，运行在 http://localhost:8080/api
- 前端：Vue 3 + Element Plus，运行在 http://localhost:3000
- 数据库：MySQL 8.0

## 3. 测试步骤

### 3.1 后端服务启动测试
1. **进入后端目录**：`cd d:\hmwl-4\backend`
2. **构建项目**：`mvn clean install`
3. **启动服务**：`mvn spring-boot:run`
4. **验证服务启动**：
   - 查看控制台输出，确认服务已成功启动
   - 检查端口 8080 是否被占用：`netstat -ano | findstr :8080`

### 3.2 前端服务启动测试
1. **进入前端目录**：`cd d:\hmwl-4\frontend`
2. **启动开发服务器**：`npm run dev`
3. **验证服务启动**：
   - 查看控制台输出，确认服务已成功启动在 http://localhost:3000
   - 浏览器访问 http://localhost:3000，确认页面加载正常

### 3.3 API接口测试
1. **测试业务用户列表接口**：
   - 执行：`curl http://localhost:8080/api/business-user/list -UseBasicParsing`
   - 预期结果：返回业务用户列表JSON数据

2. **测试业务客户列表接口**：
   - 执行：`curl http://localhost:8080/api/business-customer/listByBusinessUserId/1 -UseBasicParsing`（假设业务用户ID为1）
   - 预期结果：返回该业务用户下的客户列表JSON数据

### 3.4 功能测试
1. **访问订单管理页面**：
   - 浏览器访问 http://localhost:3000/#/order
   - 预期结果：页面加载正常，显示订单列表

2. **测试新增订单功能**：
   - 点击"新增订单"按钮
   - 选择业务用户
   - 预期结果：业务客户下拉框自动加载该业务用户的客户列表
   - 选择业务客户
   - 预期结果：收件人信息自动填充

3. **测试编辑订单功能**：
   - 选择一个订单，点击"编辑"按钮
   - 预期结果：表单自动填充订单信息，业务客户下拉框显示对应客户列表

4. **测试字符编码**：
   - 新增一个订单，填写中文收件人信息
   - 保存订单
   - 查看订单列表
   - 预期结果：收件人信息显示正常，无乱码（??）

## 4. 测试验证

### 4.1 后端验证
- 服务是否成功启动在端口 8080
- API接口是否返回正确的JSON数据
- 数据库连接是否正常
- 字符编码配置是否正确

### 4.2 前端验证
- 页面是否正常加载
- 业务用户选择后是否自动加载对应业务客户
- 业务客户选择后是否自动填充收件人信息
- 中文信息是否显示正常，无乱码

## 5. 故障排除

### 5.1 后端启动失败
- 检查端口 8080 是否被占用：`netstat -ano | findstr :8080`
- 检查数据库连接配置是否正确
- 检查依赖是否完整：`mvn dependency:tree`

### 5.2 前端代理配置错误
- 检查 vite.config.js 中的代理配置
- 确保 rewrite 规则正确：`rewrite: (path) => path.replace(/^\/api/, '')`

### 5.3 API接口访问失败
- 检查后端服务是否运行
- 检查API路径是否正确
- 检查后端控制器是否正确实现

### 5.4 业务客户列表不显示
- 检查后端 `BusinessCustomerController` 的 `listByBusinessUserId` 方法是否正确实现
- 检查前端 `getBusinessCustomerList` 方法是否正确调用API
- 检查网络请求是否成功，查看浏览器开发者工具的网络面板
