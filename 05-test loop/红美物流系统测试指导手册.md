# 红美物流系统测试指导手册

## 一、环境准备

### 1.1 启动MySQL数据库

```bash
# 启动MySQL服务
mysql.server start

# 验证MySQL是否启动成功
lsof -i :3306
```

### 1.2 启动后端服务

```bash
# 进入后端目录
cd /Users/jiangxiaochun/Desktop/hmwl/hmwl/backend

# 启动后端（后台运行）
nohup mvn spring-boot:run > /tmp/backend.log 2>&1 &

# 等待后端启动（约25秒）
sleep 25

# 验证后端是否启动成功
lsof -i :8081

# 测试后端API
curl -s http://localhost:8081/api/user/login
```

### 1.3 启动前端服务

```bash
# 进入前端目录
cd /Users/jiangxiaochun/Desktop/hmwl/hmwl/frontend

# 启动前端（后台运行）
nohup npm run dev > /tmp/frontend.log 2>&1 &

# 等待前端启动（约10秒）
sleep 10

# 验证前端是否启动成功
lsof -i :3000
```

### 1.4 访问系统

- 前端地址：http://localhost:3000
- 后端API地址：http://localhost:8081

## 三、测试流程

### 流程图

```
客户下单 → 网点报价 → 网点揽收 → 司机提货 → 运输中 → 到达网点 → 确认收货 → 调度回单确认 → 完成
```

### 3.1 客户角色测试

#### 3.1.1 登录

1. 打开浏览器访问：http://localhost:3000
2. 输入账号：customer1 / 123456
3. 点击登录

#### 3.1.2 创建订单

1. 登录后进入"客户订单"页面
2. 点击"新增订单"按钮
3. 填写订单信息：
   - 发件人：从地址簿选择（已有1618条发货人数据）
   - 收件人：从地址簿选择
   - 货物信息：填写货物名称、重量、数量
   - 备注：可选填写
4. 点击"提交"按钮

#### 3.1.3 查看订单

1. 进入"我的订单"页面
2. 查看订单状态：待报价

#### 3.1.4 API测试命令

```bash
# 客户登录
curl -X POST http://localhost:8081/api/user/login \
  -H "Content-Type: application/json" \
  -d '{"username":"customer1","password":"123456"}'

# 获取客户订单列表
curl -s "http://localhost:8081/api/customer/orders?customerId=1"

# 获取发货人地址簿
curl -s "http://localhost:8081/api/business-customer/list?businessUserId=1&type=0"
```

---

### 3.2 网点角色测试

#### 3.2.1 登录

1. 打开浏览器访问：http://localhost:3000
2. 输入账号：network1 / 123456
3. 点击登录

#### 3.2.2 报价操作

1. 登录后进入"我的订单"页面
2. 在"报价"Tab中查看待报价订单
3. 点击"报价"按钮
4. 填写报价信息：底价、最终价、运输天数
5. 点击"确定"提交报价

#### 3.2.3 揽收操作

1. 在"运输"Tab中查看待揽收订单
2. 点击"确认揽收"按钮
3. 确认揽收信息

#### 3.2.4 确认收货

1. 在"运输"Tab中查看已运输订单
2. 点击"确认收货"按钮
3. 上传回单凭证（最多3张）
4. 选择检查结果：正常/异常
5. 填写备注（可选）
6. 点击"确定"确认收货

#### 3.2.5 API测试命令

```bash
# 网点登录
curl -X POST http://localhost:8081/api/user/login \
  -H "Content-Type: application/json" \
  -d '{"username":"network1","password":"123456"}'

# 获取网点待报价订单
curl -s "http://localhost:8081/api/network/pending-quotes?networkPointId=1"

# 获取网点订单列表
curl -s "http://localhost:8081/api/order/network-list?networkPointId=1"

# 提交报价
curl -X POST http://localhost:8081/api/network/save-quote \
  -H "Content-Type: application/json" \
  -d '{"orderId":1,"networkId":1,"baseFee":10,"finalPrice":15,"transitDays":3}'

# 确认收货
curl -X POST http://localhost:8081/api/network/confirm-receive \
  -H "Content-Type: application/json" \
  -d '{"orderId":1,"networkId":1,"checkResult":"正常"}'
```

## 六、常见问题

### 6.1 后端启动失败

- 检查MySQL是否启动：lsof -i :3306
- 查看后端日志：tail -50 /tmp/backend.log

### 6.2 前端页面加载慢

- 清除浏览器缓存
- 使用无痕模式打开

### 6.3 数据库乱码

- 导入数据时指定字符集：
```bash
mysql -uroot --default-character-set=utf8mb4 hmwl < 文件.sql
```

### 6.4 订单状态不更新

- 检查后端API是否正常响应
- 查看订单状态字段值
